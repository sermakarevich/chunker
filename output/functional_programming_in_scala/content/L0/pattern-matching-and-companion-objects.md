# pattern-matching-and-companion-objects

**Parent:** [[content/L1/functional-data-structures-scala|functional-data-structures-scala]] — In Scala, functional data structures are immutable, utilizing the `sealed trait List[+A]` defined with `Nil` and `Cons(head: A, tail: List[A])`. Functions like `sum` (summing `List[Int]` by matching `Nil => 0` and `Cons(x,xs) => x + sum(xs)`) and `product` (handling `Nil => 1.0` and `Cons(0.0, _) => 0.0`) illustrate recursive pattern matching, while the `foldRight` higher-order function provides a generalized method for list operations, such as calculating $1+2+3=6$ via `foldRight(Cons(1, Cons(2, Cons(3, Nil))), 0)((x, y) => x + y)`.

Within Scala, a data type's instance can be considered a subtype of another data type's instance if the contained types satisfy a subtype relationship. For example, if `Dog` is a subtype of `Animal`, then `List[Dog]` is considered a subtype of `List[Animal]`. This principle generalizes: for any types X and Y, if X is a subtype of Y, then `List[X]` is a subtype of `List[Y]`. The original type parameter definition, `sealed trait List[+A]`, uses `+A` to indicate that A is a covariant parameter. Using this annotation is beneficial because `Nil` extends `List[Nothing]`, and since `Nothing` is a subtype of all types, `Nil` can therefore be treated as a `List[Int]`, a `List[Double]`, and so on, fulfilling the desired behavior.

In addition to variance considerations, functional data structures can utilize pattern matching. The functions `sum` and `product`, placed within the `List` companion object, demonstrate this. The `sum` function, which accepts `List[Int]` arguments, is defined as: `def sum(ints: List[Int]): Int = ints match { case Nil => 0; case Cons(x,xs) => x + sum(xs) }`. This implementation states that the sum of an empty list (`Nil`) is 0, and the sum of a non-empty list is the first element (`x`) plus the sum of the remaining elements (`xs`). Similarly, the `product` function, which accepts `List[Double]` arguments, is defined as: `def product(ds: List[Double]): Double = ds match { case Nil => 1.0; case Cons(0.0, _) => 0.0; case Cons(x,xs) => x * product(xs) }`. These definitions are recursive, a common pattern when writing functions that operate over recursive data types like `List`, which refers to itself recursively in its `Cons` data constructor.

Pattern matching functions like `sum` and `product` work by descending into the structure of the provided expression (the target or scrutinee). The process begins with an expression like `ds` followed by the `match` keyword and a sequence of curly brace `{}` wrapped cases. Each case consists of a pattern (e.g., `Cons(x,xs)`) to the left of the `=>` and a result (e.g., `x * product(xs)`) to the right. If the target matches a pattern, that case's result is the outcome of the entire match expression. If multiple patterns match the target, Scala selects the first matching case.

**Companion Objects:** A common practice is to declare a companion object alongside a data type and its data constructors. This companion object, sharing the same name as the data type (e.g., `List`), holds various utility functions for working with instances of the data type. Companion objects are a convention in Scala. The `List` companion object, for example, can house a function `def fill[A](n: Int, a: A): List[A]` that constructs a list containing `n` copies of element `a`.

**Pattern Matching Examples:**
* Using the variable pattern `_` matches any expression. For example, `List(1,2,3) match { case _ => 42 }` results in 42.
* Using a data constructor pattern to bind a subexpression: `List(1,2,3) match { case Cons(h,_) => h }` results in 1, where `h` is the head of the list.
* Extracting the tail: `List(1,2,3) match { case Cons(_,t) => t }` results in `List(2,3)`. If `List(1,2,3) match { case Nil => 42 }` is executed, a `MatchError` occurs because none of the cases match the target.

**Matching a list structure:** When matching a list like `Cons(1, Cons(2, Cons(3, Nil)))`, the code structure is: `List(1, 2, 3) match { case Cons(h, _) => h }` results in 1. The underscore variable pattern (`_`) can be used multiple times within a pattern to ignore multiple parts of the target.
