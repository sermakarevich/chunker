# parallel-map-and-filter-implementation

**Parent:** [[content/L1/functional-programming-advanced-combinators|functional-programming-advanced-combinators]] — Parallel computations require combinators like `parMap` (`def parMap[A,B](ps: List[A])(f: A => B): Par[List[B]]`) and `parFilter` (`def parFilter[A](as: List[A])(f: A => Boolean): Par[List[A]]`), which must be implemented using established primitives like `asyncF(f)` and `sequence` to manage complex asynchronous logic and ensure proper adherence to timeout requirements.

The process of implementing `parMap` requires recognizing that defining `parMap` as a new primitive combinator is complex, especially when ensuring proper adherence to timeout requirements. The function signature for `parMap` is `def parMap[A,B](ps: List[A])(f: A => B): Par[List[B]]`. Implementing `parMap` involves two necessary steps. First, developers must map the input list `ps` to a list of parallel computations (`List[Par[B]]`) by utilizing the `asyncF(f)` combinator. The `asyncF` combinator converts a standard function `A => B` into a function with the signature `A => Par[B]`, which subsequently permits the easy generation of $N$ parallel computations. Second, developers must convert the resultant `List[Par[B]]` into the `Par[List[B]]` required by the `parMap` return type. This necessity mandates defining a function called `sequence`, which has the signature `def sequence[A](ps: List[Par[A]]): Par[List[A]]`. Using these combinators, the concrete implementation for `parMap` is: `def parMap[A,B](ps: List[A])(f: A => B): Par[List[B]] = fork { val fbs: List[Par[B]] = ps.map(asyncF(f)); sequence(fbs) }`. When executing this implementation, `parMap` returns immediately, even if the input list `ps` is very large. However, when a user later calls `run`, the system executes a single asynchronous computation. This computation then spawns $N$ parallel computations and subsequently waits for all of these individual computations to finish, collecting their results into a list. The next task is to implement `parFilter`, which filters elements of a list in parallel; this function has the signature `def parFilter[A](as: List[A])(f: A => Boolean): Par[List[A]]`. Utilizing established primitive combinators for `parFilter` is advantageous because it prevents the duplication of complex logic, particularly when respecting timeout requirements. Finally, readers are encouraged to experiment by writing other useful parallel computations, suggesting three specific tasks: 1) creating a more general parallel summation function (discussed earlier in the chapter) and applying it to find the maximum value of an `IndexedSeq` in parallel; 2) writing a function that takes a list of paragraphs (`List[String]`) and returns the total number of words across all paragraphs in parallel, generalizing this function as much as possible; and 3) implementing `map3`, `map4`, and `map5` using the existing `map2` combinator.
