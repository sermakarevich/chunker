# functional-data-structure-list-recursion

**Parent:** [[content/L1/scala-functional-data-structures-and-higher-order-functions|scala-functional-data-structures-and-higher-order-functions]] — The comprehensive synthesis details advanced Scala concepts, including persistent functional data structures, the non-tail-recursive nature of `foldRight` (which causes stack overflows on large inputs), and the necessity of implementing tail-recursive `foldLeft` for large lists. It also covers advanced function generalization like `flatMap` and `zipWith`, and modern type-safe error handling using the `Option[+A]` ADT instead of exceptions or sentinel values.

When implementing functional data structures, the `foldRight` function must traverse the entire list, which pushes frames onto the call stack, before it can begin collapsing the results. Regarding anonymous functions, the syntax `(x,y) => x + y` can be replaced by `_ + _` in Scala if the types of the parameters `x` and `y` can be inferred from the context. This shorthand is useful when the function parameters are referenced only once within the function body. The Scala standard library provides `foldRight` as a method on `List`, and its arguments are curried for enhanced type inference. 

Developers can use the syntax `_ + _` for addition, `x => x * 2` for multiplication, or `_ * 2` for other operations. Other examples include `_.head` and `(xs,n) => xs.drop(n)`. However, users must exercise caution because the meaning of this underscore-based anonymous function syntax in complex expressions, such as `foo(_, g(List(_ + 1), _))`, can become unclear. Because precise rules govern the scoping of these underscore-based anonymous functions in the Scala language specification, the authors recommend using ordinary named function parameters when complexity is involved.

**Standard Library Methods:**
The `List` type in the standard library uses `::` (which associates to the right) as the list constructor, meaning `1 :: 2 :: Nil` equals `1 :: (2 :: Nil)`, which is equivalent to `List(1,2)`. When pattern matching, the case `Cons(h,t)` becomes `h :: t`, which prevents the need for nested parentheses when writing a pattern like `case h :: h2 :: t` to extract more than just the first element of the list. The standard library provides several other methods on `List[A]`: `take(n: Int)` (returns the first `n` elements), `takeWhile(f: A => Boolean)` (returns the longest valid prefix of the list whose elements all satisfy the predicate `f`), `forall(f: A => Boolean)` (returns `true` if and only if all elements pass the predicate `f`), and `exists(f: A => Boolean)` (returns `true` if any element passes the predicate `f`). Additionally, `scanLeft` and `scanRight` return the list of partial results, similar to how `foldLeft` and `foldRight` return only the final accumulated value.
