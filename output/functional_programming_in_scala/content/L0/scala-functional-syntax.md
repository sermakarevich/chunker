# scala-functional-syntax

**Parent:** [[content/L1/functional-programming-purity-side-effects|functional-programming-purity-side-effects]] — The initial impure `buyCoffee` method charges the credit card directly, an observable side effect, which is refactored into the pure `buyCoffees` function that returns a `Charge` data type, enabling composable and testable logic.

The `abs` method serves as a pure function, taking an integer (`Int`) as input and returning its absolute value. This method is defined as: `def abs(n: Int): Int = if (n < 0) -n else n`. Here, the `def` keyword initiates the method name, followed by the parameter list in parentheses; in this case, `abs` takes a single argument, `n`, of type `Int`. After the closing parenthesis of the argument list, the optional type annotation (`: Int`) specifies that the result type is `Int` (the colon is pronounced “has type”). The method body follows a single equals sign (`=`). The part of a declaration before the equals sign is referred to as the left-hand side or signature, and the code following the equals sign is the right-hand side or definition. Note the absence of an explicit return keyword; the value returned from a method is the result of evaluating the right-hand side. All expressions, including `if` expressions, produce a result. In this example, the right-hand side is a single expression whose value is either `-n` or `n`, depending on whether `n < 0`. 

The `formatAbs` method is another pure function: `private def formatAbs(x: Int) = { val msg = "The absolute value of %d is %d." msg.format(x, abs(x)) }`. The `format` method is a standard library method defined on the `String` class. In this method, the code calls `format` on the `msg` object, passing the value of `x` and the value of `abs` applied to `x`. This results in a new string where the occurrences of `%d` in `msg` are replaced by the evaluated results of `x` and `abs(x)`, respectively. The `formatAbs` method is declared `private`, meaning that it cannot be called from any code outside of the `MyModule` object. Although this function takes an `Int` and returns a `String`, the return type is not explicitly declared because Scala is usually capable of inferring method return types, though explicitly declaring them is considered good style for methods expected by others. Since this method is private to the module, the explicit type annotation can be omitted. Because the method body contains more than one statement, the statements are enclosed within curly braces, which defines a block. Statements within a block are separated by newlines or semicolons; here, a newline is used, so a semicolon is not necessary. The first statement in the block declares an immutable variable (`val`) named `msg` using the `val` keyword. This declaration simply assigns a name to the string value so the program can refer to it again. Since `val` is an immutable variable, inside the body of `formatAbs`, the name `msg` will always refer to the same `String` value, and the Scala compiler will complain if the programmer attempts to reassign `msg` to a different value in the same block. Remember that a method returns the value of its right-hand side, eliminating the need for a return keyword. In this case, the right-hand side is a block. In Scala, the value of a multi-statement block inside curly braces is the same as the value returned by the last expression in the block. Therefore, the result of the `formatAbs` method is simply the value returned by the call to `msg.format(x, abs(x))`. 

Finally, the `main` method serves as the outer shell, calling into the purely functional core and printing the result to the console. Programmers sometimes call such methods procedures (or impure functions) to emphasize that they have side effects: `def main(args: Array[String]): Unit = println(formatAbs(-42))`. The name `main` is special because when running a program, the Scala compiler will look for a method named `main` with a specific signature. This method must accept an `Array` of `String` as its argument, and its return type must be `Unit`. The `args` array contains the arguments provided at the command line; these arguments are not used in this program. `Unit` is the special type that serves a similar purpose to `void` in languages like C and Java. Because every method in Scala must return a value unless it crashes or hangs, `Unit` is required for methods like `main` that do not return meaningful values. The literal syntax for `Unit` is `()`, a pair of empty parentheses. A return type of `Unit` is usually a hint that the method has a side effect. The body of the `main` method uses `println` to print the `String` returned by the call to `formatAbs`. This printing action uses `println`, which itself has a return type of `Unit`, matching the required return type for `main`. 

Regarding program execution, the simplest way to run the Scala program (`MyModule`) is by invoking the Scala compiler directly from the command line. By saving the code in a file named `MyModule.scala`, the programmer can compile it using `scalac MyModule.scala`, which generates files ending with the `.class` suffix, containing code executable by the Java Virtual Machine (JVM). Alternatively, for simple programs, the code can be run directly using the Scala interpreter by passing it to the `scala` command-line tool: `scala MyModule.scala`. When using the Scala interpreter, the tool searches for any object within `MyModule.scala` that possesses a `main` method with the appropriate signature and calls it. Another alternative is starting the Scala interpreter's interactive mode, the REPL (read-evaluate-print loop). Users can load the source file into the REPL using the command `scala> :load MyModule.scala`, which evaluates the Scala expression and prints the result, assigning it a temporary name (e.g., `res0: Int = 42`). 

When discussing modular structure, the `MyModule` object acts as a namespace, as demonstrated by the necessity to reference the `abs` method using `MyModule.abs`, since `abs` was defined within the `MyModule` object. In Scala, every value is considered an object, and each object may contain zero or more members. An object whose primary function is to provide a namespace for its members is often called a module. Members can be declared using the `def` keyword (methods) or using `val` or `object` (other objects). Programmers access the members of these objects using the standard object-oriented dot notation: `namespace.member`.
