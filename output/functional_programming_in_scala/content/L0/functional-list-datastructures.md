# functional-list-datastructures

**Parent:** [[content/L1/functional-programming-principles|functional-programming-principles]] — Functional programming ensures type safety and non-mutation using patterns like Option[+A] and Either[+A, +B] for explicit error handling, and employs techniques like non-strictness (laziness) to achieve automatic loop fusion and optimize data transformations by avoiding intermediate lists. To maintain referential transparency, mutable state, such as random number generation, must be managed by defining an interface (e.g., `RNG`) that forces methods to return both the resulting value and the new state, rather than using side effects.

In functional programming, data structures are fundamentally immutable and can only be operated on using pure functions, meaning the data structures cannot change data in place or perform any side effects. For instance, concatenating two lists using the syntax `a ++ b` yields a new list without modifying the original lists, $a$ and $b$. The core focus is on the singly linked list, which is the `List` data type in Scala's standard library. To define this type, a trait, `sealed trait List[+A]`, is used, and `sealed` ensures that all implementations of the trait must be declared in the file. Two data constructors define the two possible list forms: the empty list, denoted by the case object `Nil` (which extends `List[Nothing]`), or a non-empty list, denoted by the case class `Cons[+A](head: A, tail: List[A])` (which extends `List[A]`). The type parameter `+A` indicates that A is a covariant parameter. While the principles of data sharing demonstrate that adding or removing elements (e.g., creating `Cons(1, xs)` or returning `xs` when defined as `Cons(x, xs)`) reuses the original list, thus making the structures **persistent**, there are exceptions to efficiency. Specifically, the `init` function, which must return a list consisting of all but the last element, cannot be implemented in constant time like `tail` because the single linked list structure requires copying all preceding `Cons` objects when the tail must be replaced. 

Developers can utilize advanced techniques like `foldRight` to generalize list operations. The signature for `foldRight` is `def foldRight[A, B](as: List[A], z: B)(f: (A, B) => B): B`. This function processes a list by pattern matching, using `z` as the initial value and `f` (a function) to combine the list element (`A`) with the accumulated result (`B`). For example, calculating `sum2` involves calling `foldRight(ns, 0)((x, y) => x + y)`, where 0 is the initial value and the lambda function `(x, y) => x + y` defines the addition operation.

To improve efficiency and modularity, the programming technique of automatic loop fusion—which involves fusing sequences of transformations—is achieved using non-strictness, or laziness. Non-strictness is a property of a function, meaning the function may choose not to evaluate one or more of its arguments. By default, any function in Scala, including all previously defined functions, is strict. For instance, calling `square(41.0 + 1.0)` yields `square` receiving the fully evaluated value of `42.0`.
