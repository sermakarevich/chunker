# property-based-testing-api-refinement

**Parent:** [[content/L1/pbt-dependency-refinement-and-parallelism-testing|pbt-dependency-refinement-and-parallelism-testing]] — The Prop structure was refined to accept `MaxSize` in addition to `TestCases` and `RNG`, changing the signature from `(TestCases, RNG) => Result` to `(MaxSize, TestCases, RNG) => Result`. This adaptation was necessary to correctly structure property generation for `forAll` and ensure proper handling of constraints like non-empty lists and parallel computations.

The initial approach to defining the `forAll` function, which accepts a generator `Gen[A]` and a predicate `A => Boolean` to create a property, was found insufficient because the `SGen` type expected a size parameter, but the `Prop` definition did not receive any size information. To resolve this dependency, the definition of `Prop` was updated to accept a maximum size, treating this size as a required dependency, similar to the dependency on the source of randomness and the number of test cases. Consequently, the `Prop` case class was updated to `case class Prop(run: (MaxSize, TestCases, RNG) => Result)`, where `MaxSize` is defined as `type MaxSize = Int`. The `forAll` function was consequently updated to match this structure, defining it as `def forAll[A](g: Int => Gen[A])(f: A => Boolean): Prop = Prop { (max, n, rng) => ... }`. This revised implementation structures the generation process by iterating over sizes from 0 up to $\min(n, max) + 1$ (where $n$ is the initial specified size and $max$ is the maximum size). For each size $i$, the function generates a set of random cases and subsequently combines them into a single property (`prop`) using the $\&\&$ operator. The final function execution is performed by calling `prop.run(max, n, rng)`.

To improve the usability of the library, a helper function, `run`, was introduced as a method on the `Prop` companion object. This `run` function simplifies testing by taking a `Prop` value and printing the result to the console in a readable format. The `run` function accepts default arguments for `maxSize` (defaulting to 100), `testCases` (defaulting to 100), and `rng` (defaulting to `RNG.Simple(System.currentTimeMillis)`). The function calls `p.run(maxSize, testCases, rng)` and uses pattern matching on the returned `Result` to either print the failure details (`Falsified(msg, n)`) to the console or print an OK message for successful execution (`Passed`).

By applying this enhanced library structure to specify the behavior of the `max` function (which finds the maximum value of a list, available on `List` via API documentation at `http://mng.bz/Pz86`), the example property `maxProp` was constructed. This property takes a list of randomly generated small integers (generated by `val smallInt = Gen.choose(-10, 10)`) and asserts that the maximum value of the list must be greater than or equal to every other element in the list, checking the condition `!ns.exists(_ > max)` where `max = ns.max`. Initial testing using `run(maxProp)` revealed a failure because the standard library's implementation of `max` crashes when given an empty list, necessitating a correction to the property definition to account for this limitation.
