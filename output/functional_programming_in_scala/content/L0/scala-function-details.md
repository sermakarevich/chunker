# scala-function-details

**Parent:** [[content/L1/functional-programming-purity-side-effects|functional-programming-purity-side-effects]] — The initial impure `buyCoffee` method charges the credit card directly, an observable side effect, which is refactored into the pure `buyCoffees` function that returns a `Charge` data type, enabling composable and testable logic.

The `abs` method is a pure function that takes an integer (`Int`) as input and returns the input's absolute value. The method is defined using the following syntax: `def abs(n: Int): Int = if (n < 0) -n else n`. In this definition, the `def` keyword initiates the method name, followed by the parameter list enclosed in parentheses. Specifically, `abs` takes a single argument, `n`, which is of type `Int`. Following the closing parenthesis of the argument list, the optional type annotation (`: Int`) specifies that the result type is `Int` (which means the return type is `Int`). The method body is initiated by a single equals sign (`=`). The part of a declaration that appears before the equals sign is called the left-hand side or signature, while the code following the equals sign is the right-hand side or definition. It is notable that the method does not use an explicit return keyword, as the value returned from a method is automatically determined by the result of evaluating the right-hand side. All expressions, including `if` expressions, produce a result. In this specific example, the right-hand side is a single expression whose resulting value will be either `-n` or `n`, depending on the boolean evaluation of whether `n < 0`. 

The `formatAbs` method is another pure function defined as: `private def formatAbs(x: Int) = { val msg = "The absolute value of %d is %d." msg.format(x, abs(x)) }`. The `format` method used within `formatAbs` is a standard library method defined on the `String` class. Within the body of `formatAbs`, the code calls `format` on the `msg` object, passing the value of `x` and the value of `abs` applied to `x`. This process generates a new string where the occurrences of `%d` in `msg` are replaced, respectively, by the evaluated results of `x` and `abs(x)`. The `formatAbs` method is declared as `private`, meaning that any code outside of the `MyModule` object cannot call it. Although this function accepts an `Int` and returns a `String`, the return type is not explicitly declared because Scala can usually infer method return types, though explicit declarations are considered good style for methods intended for use by others. Since this method is private to the module, the explicit type annotation can be omitted. Because the method body contains more than one statement, the statements must be enclosed within curly braces, which define a block. Statements within a block are separated by newlines or semicolons; here, a newline is used, so a semicolon is not necessary. The first statement in the block declares an immutable variable, or `val`, named `msg` using the `val` keyword. This declaration simply assigns a name to the string value so that the program can reference it again. Since `val` defines an immutable variable, inside the body of `formatAbs`, the name `msg` will always refer to the same `String` value, and the Scala compiler will issue a complaint if a programmer attempts to reassign `msg` to a different value within the same block. The `formatAbs` method follows the rule that a method returns the value of its right-hand side, thereby eliminating the need for a return keyword. In this case, the right-hand side is a block. In Scala, the value of a multi-statement block enclosed within curly braces is equivalent to the value returned by the last expression in the block. Therefore, the final result of the `formatAbs` method is simply the value returned by the call to `msg.format(x, abs(x))`. 

Finally, the `main` method acts as the outer shell, calling into the purely functional core and printing the resulting string to the console. Programmers sometimes refer to such methods as procedures (or impure functions) to emphasize that they perform side effects: `def main(args: Array[String]): Unit = println(formatAbs(-42))`. The name `main` is special because the Scala compiler looks for a method named `main` with a specific signature when running a program. This method must accept an `Array` of `String` as its argument, and its required return type must be `Unit`. The `args` array contains command line arguments, which are not utilized in this program. `Unit` is the special type serving a purpose similar to `void` in languages such as C and Java. Because every method in Scala must return a value unless it crashes or hangs, `Unit` is required for methods like `main` that do not return meaningful values. The literal syntax for `Unit` is `()` (a pair of empty parentheses). A return type of `Unit` typically suggests that the method has a side effect. The body of the `main` method uses `println` to print the `String` that is returned by the call to `formatAbs`. This printing action uses `println`, which itself has a return type of `Unit`, matching the required return type for `main`. 

Regarding program execution, the simplest way for a programmer to run the Scala program (`MyModule`) is by invoking the Scala compiler directly from the command line. By saving the code in a file named `MyModule.scala`, the programmer can compile it using `scalac MyModule.scala`, which generates files with the `.class` suffix, containing code executable by the Java Virtual Machine (JVM). Alternatively, for simple programs, the code can be run directly using the Scala interpreter by passing it to the `scala` command-line tool: `scala MyModule.scala`. When using the Scala interpreter, the tool searches for any object within `MyModule.scala` that possesses a `main` method with the appropriate signature and calls it. Another alternative is starting the Scala interpreter's interactive mode, the REPL (read-evaluate-print loop). Users can load the source file into the REPL using the command `scala> :load MyModule.scala`, which evaluates the Scala expression and prints the result, assigning it a temporary name (e.g., `res0: Int = 42`). 

When discussing modular structure, the `MyModule` object functions as a namespace, which is demonstrated by the necessity of referencing the `abs` method using `MyModule.abs`, since `abs` was defined within the `MyModule` object. In Scala, every value is considered an object, and each object may contain zero or more members. An object whose primary function is providing a namespace for its members is often called a module. Members can be declared using the `def` keyword (methods) or using `val` or `object` (other objects). Programmers access the members of these objects using the standard object-oriented dot notation: `namespace.member`.
