# functional-programming-scala-guide

**Parent:** [[content/L1/functional-programming-in-scala-context|functional-programming-in-scala-context]] — The book *Functional Programming in Scala* by Paul Chiusano and Rúnar Bjarnason, copyrighted in 2015 and published by Manning Shelter Island, is structured into four parts (starting on pages 1, 93, 173, and 227, respectively) and teaches FP principles by treating Scala as a purely functional language, detailing core concepts, functional design patterns, advanced abstractions, and finally, real-world I/O operations.

What is functional programming? Fundamentally, functional programming (FP) is a programming style that focuses on functions within a program, sometimes simply referred to as “programming with functions.” The definition of functions varies, but pure functional programming restricts functions to behave as they do in mathematics: binary relations that map arguments to results. While one definition of functions may admit side effects in addition to returning a result, pure functional programming demands that functions remain pure. Scala is considered an impure functional programming language because it admits both impure and pure functions, and it does not distinguish between these two categories using different syntax or types. Scala shares this property with most other functional languages, though the author suggests it would be beneficial to find a way to distinguish pure and impure functions within Scala that is lightweight and flexible enough for easy adoption.

Despite these challenges, Scala programmers are generally encouraged to utilize pure functions. Side effects, such as mutation, I/O operations, or the use of exceptions, are not ruled out and can be useful for reasons of interoperability, efficiency, or convenience. However, experts generally consider overusing side effects to be poor programming style. Because impure programs are possible and convenient to write in Scala, programmers coming from an imperative background may be tempted to maintain their existing style instead of making the necessary effort to adapt to the functional mindset; in fact, it is quite possible to write Scala as if it were Java but without semicolons.

To learn functional programming properly using Scala, the author questions whether one should take a detour through a pure functional language like Haskell. However, any argument favoring such a detour has been significantly weakened by the emergence of Functional Programming in Scala. Paul and Rúnar address this by treating Scala as if it were a pure functional programming language, eliminating mutable variables, exceptions, classical input/output, and all other signs of impurity. The authors demonstrate, starting from first principles and extending to incremental input and output, that every concept can be expressed using only pure functions. Furthermore, they show that this approach is not only possible but also leads to beautiful code and deep insights into the nature of computation.

Paul Chiusano and Rúnar Bjarnason’s book is described as challenging, requiring attention to detail and potentially challenging the reader’s existing views on programming. The book recommends that readers benefit from reading the material and completing the suggested exercises, which will help develop a better appreciation for what pure functional programming is, what it can express, and its benefits. The author praises the book for being self-contained, starting with the simplest expressions, and explaining every abstraction in detail before building further abstractions upon them. In essence, the book develops an alternative Scala universe where mutable state does not exist and all functions are pure. Although commonly used Scala libraries often deviate from this ideal—frequently relying on partly imperative implementations with a (mostly) functional interface—the ability of Scala to encapsulate mutable state in a functional interface is noted as a strength, though the author warns that this capability is often misused, and *Functional Programming in Scala* serves as a powerful remedy.

**Prefix (Preface & Acknowledgments):**
Both the authors, Martin Odersky and the unnamed co-author, had encountered difficulty when first learning functional programming (FP). They recognized that the FP approach, despite being different, leads to a coherent, composable, and beautiful way of writing programs. They had both learned FP somewhat haphazardly by writing functional code, talking to and learning from other Scala and Haskell programmers, and reading various fragmented resources. In April 2010, Nermin Šerifović, an organizer in the Boston Area Scala Enthusiasts group, suggested the authors write a book specifically on FP in Scala. After more than four years of work, the authors feel they have created a comprehensive resource, one they wished had existed when they initially learned functional programming.

**About This Book (Structure):**
The book is intended as an introduction to functional programming (FP), a radical, principled approach to writing software. While Scala is used as the vehicle for instruction, the lessons are applicable to programming in any language. The book is organized into four parts:
1. **Part 1:** Discusses the core concepts of functional programming, giving an overview of fundamental techniques such as organizing small functional programs, defining purely functional data structures, handling errors, and managing state.
2. **Part 2:** Consists of tutorials on functional design, working through practical functional libraries to illustrate the underlying thought process for designing them.
3. **Part 3:** Introduces higher abstractions needed for writing more generalized libraries, highlighting patterns and duplication noticed while working through the libraries in Part 2. These abstractions are presented as powerful tools for reasoning about code and promise extraordinary programmer productivity upon mastery.
4. **Part 4:** Bridges the gap toward writing real-world applications that perform I/O (e.g., working with databases, files, or video displays) and utilize mutable state, all within a purely functional methodology.

Throughout the book, the authors rely heavily on carefully sequenced programming exercises, emphasizing that understanding functional programming requires more than just abstract theory; readers must actively write code.
