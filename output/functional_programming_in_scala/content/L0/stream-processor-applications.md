# stream-processor-applications

**Parent:** [[content/L1/advanced-functional-architecture-overview|advanced-functional-architecture-overview]] — The synthesis describes a robust functional architecture relying on Algebraic Data Types (ADTs) and higher-order functions to achieve referential transparency. Key mechanisms include `Stream[+A]` for memory-efficient lazy data processing, `Option` and `Either` for controlled flow, `Par[A]` and the Actor model for non-blocking concurrency, and the `Free[F[_],A]` type for generalized effect modeling, exemplified by `IO` using trampolining.

While the previous examples focused on processing text line-by-line, the same stream processor library can be utilized for processing binary files. Stream processors are highly valuable for modeling large-scale systems, such as message processing, state machines, and actors. These large systems are frequently structured as loosely coupled components that communicate using explicit message sending and receiving, often modeled using actors. Developers can represent these complex components and their interactions as stream processors, which allows for describing complex state machines and behaviors using a high-level, compositional API. Furthermore, web applications can be conceptualized as transforming a stream of HTTP requests into a corresponding stream of HTTP responses. Similarly, user interface (UI) programming allows viewing individual events, such as mouse clicks, as streams, enabling the entire UI to be modeled as a large network of stream processors that determine how the UI reacts to user interaction. In the domain of big data and distributed systems, stream-processing libraries support distribution and parallelization when processing massive amounts of data; this means the physical nodes of a stream-processing network do not need to reside on the same machine. For additional information on these applications, or others, the chapter notes provide further discussion and links. The chapter notes and associated code also detail extensions to the `Process` type, including the introduction of nondeterministic choice, which permits concurrent evaluation when executing a `Process`.
