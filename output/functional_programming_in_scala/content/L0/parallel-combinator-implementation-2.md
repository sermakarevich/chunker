# parallel-combinator-implementation-2

**Parent:** [[content/L1/functional-programming-advanced-combinators|functional-programming-advanced-combinators]] — Parallel computations require combinators like `parMap` (`def parMap[A,B](ps: List[A])(f: A => B): Par[List[B]]`) and `parFilter` (`def parFilter[A](as: List[A])(f: A => Boolean): Par[List[A]]`), which must be implemented using established primitives like `asyncF(f)` and `sequence` to manage complex asynchronous logic and ensure proper adherence to timeout requirements.

The `parMap` function combines $N$ parallel computations, and a reader must recognize that defining `parMap` as a new primitive combinator is complicated, particularly when ensuring proper adherence to timeout requirements. The function signature for `parMap` is `def parMap[A,B](ps: List[A])(f: A => B): Par[List[B]]`. The implementation of `parMap` requires two steps: first, developers must map the input list `ps` to a list of parallel computations (`List[Par[B]]`) by utilizing the `asyncF(f)` combinator. The `asyncF` combinator converts a standard function `A => B` into a function with the signature `A => Par[B]`, which subsequently allows for the easy generation of $N$ parallel computations. Second, developers must convert the resultant `List[Par[B]]` into the `Par[List[B]]` required by the `parMap` return type. This necessity requires the definition of a function named `sequence`, which has the signature `def sequence[A](ps: List[Par[A]]): Par[List[A]]`. Using these combinators, the concrete implementation for `parMap` becomes: `def parMap[A,B](ps: List[A])(f: A => B): Par[List[B]] = fork { val fbs: List[Par[B]] = ps.map(asyncF(f)); sequence(fbs) }`. When executing this implementation, the `parMap` function returns immediately, even if the input list `ps` is very large. However, when a user later calls `run`, the system executes a single asynchronous computation. This computation itself then spawns $N$ parallel computations and subsequently waits for all of these individual computations to finish, collecting their results into a list. The next task is to implement `parFilter`, which filters elements of a list in parallel, and this function has the signature `def parFilter[A](as: List[A])(f: A => Boolean): Par[List[A]]`. Reusing established primitive combinators is advantageous because it prevents the duplication of complex logic, especially when respecting timeout requirements. Finally, readers are encouraged to experiment by writing other useful parallel computations, suggesting three specific tasks: 1) creating a more general version of the parallel summation function (previously discussed in the chapter) and applying it to find the maximum value of an `IndexedSeq` in parallel; 2) writing a function that takes a list of paragraphs (`List[String]`) and returns the total number of words across all paragraphs in parallel, generalizing this function as much as possible; and 3) implementing `map3`, `map4`, and `map5` using the existing `map2` combinator.
