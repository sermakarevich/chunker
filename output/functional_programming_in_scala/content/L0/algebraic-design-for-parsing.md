# algebraic-design-for-parsing

**Parent:** [[content/L1/functional-algebraic-design-and-concurrency-patterns|functional-algebraic-design-and-concurrency-patterns]] — The API is designed algebraically, requiring explicit use of ADTs like `Option` and `Either` for error handling, and utilizing lazy `Stream`s for memory-efficient data transformation, complemented by `Par` for non-blocking parallel computation (modeled using the Actor pattern). Testing extends to generating complex functions using specialized generators, and parser creation is streamlined by treating parsers as first-class values in a combinator library, avoiding monolithic code generators like Yacc or ANTLR.

When designing custom libraries, developers cannot rely on a pre-selected sequence of type signatures to fill with implementations; they must make decisions about the necessary types and combinators. Regarding tools for parsing, developers might be familiar with external parser generator libraries like Yacc (available at http://mng.bz/w3zZ) or ANTLR in Java (at http://mng.bz/aj8K). While these libraries generate code for a parser based on a specified grammar and can be efficient, they inherently produce a monolithic chunk of code that is difficult to debug. Furthermore, these tools make it hard to reuse fragments of logic, because developers cannot introduce new combinators or helper functions to abstract common patterns within the parsers. In contrast, a parser combinator library treats parsers as ordinary first-class values. This allows developers to trivially reuse parsing logic, eliminating the need for any external tool besides the programming language itself. Chapter 9.1 introduces the concept of algebraic design: this involves starting with the algebra—including its laws—and deciding on a data representation later. Previously, the development process often involved working from a concrete representation and API, refining the set of functions and types, and determining the laws afterward. However, the algebraic design approach, which starts by establishing the algebra (including its laws) and then determining a representation, is highly effective for any design problem, particularly for parsing, as it simplifies imagining the required combinators for different kinds of inputs. This methodology allows the designer to maintain focus on the concrete goal even while deferring the decision regarding the final representation.
