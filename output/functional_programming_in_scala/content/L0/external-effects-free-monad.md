# external-effects-free-monad

**Parent:** [[content/L1/advanced-fp-algebraic-design|advanced-fp-algebraic-design]] — Advanced functional programming mandates using algebraic structures (ADTs, Functor, Applicative, Monad) to treat computation as a first-class value, ensuring type safety and referential transparency by encapsulating side effects. Effect management is achieved using the `Free[F, A]` monad, where $F$ is the protocol, enabling interpreters (e.g., `runConsolePar`) to translate abstract I/O (like `ConsoleIO[A]`) into concrete structures (`Par[A]`) suitable for both trampolined and non-blocking asynchronous execution.

To manage external effects and achieve functional purity, the `Free[F[_],A]` structure is used, where the type parameter $F$ acts as an executable protocol. When limiting the system to console I/O, a specialized type `ConsoleIO[A]` is defined as `Free[Console, A]`. The `Console` type contains two core instructions: `PrintLine` and `ReadLine`. For example, the method `readLn` returns a `ConsoleIO[Option[String]]` by invoking `Suspend(ReadLine)`, and the method `printLn(line: String)` returns a `ConsoleIO[Unit]` by invoking `Suspend(PrintLine(line))`. Using `ConsoleIO[A]`, developers can write programs that interact only with the console, such as: `val f1: Free[Console, Option[String]] = for { _ <- printLn("I can only interact with the console."); ln <- readLn } yield ln`. Note that the methods `readLn` and `printLn` refer to the monadic methods defined within the `Console` type, not Scala's standard `readLine` or `println`. Since the `Console` type does not form a Monad, the mechanism for execution involves translating the `Console` type into another type $G$ that does form a Monad, such as `Function0` or `Par`. This translation is accomplished by defining a `Translate[F[_], G[_]]` trait and type alias (`F ~> G`). Specific translation instances are provided, such as `consoleToFunction0`, which maps `Console[A]` to a `Function0[A]` by accessing `a.toThunk`, and `consoleToPar`, which maps `Console[A]` to a `Par[A]` by accessing `a.toPar`. Using these translations, a generalized function `runFree[F[_],G[_],A]` is introduced. This function interprets a `Free[F,A]` program by taking the translation mechanism `t: F ~> G` and an implicit `Monad[G]` instance. The interpretation proceeds through pattern matching on the structure of the `Free[F,A]` program: if the program is `Return(a)`, it yields `G.unit(a)`; if it is `Suspend(r)`, it translates and executes the resumption function `r` using `t(r)`; if it is `FlatMap(Suspend(r),f)`, it recursively interprets the subsequent computation structure `f` using the result of the translated continuation `t(r)`. This enables the implementation of convenience functions `runConsoleFunction0[A]`, which converts `Free[Console,A]` to `() => A` using `runFree` and `consoleToFunction0`, and `runConsolePar[A]`, which converts `Free[Console,A]` to `Par[A]` using `runFree` and `consoleToPar`. These conversions require `Monad[Function0]` and `Monad[Par]` instances, which define how `unit` and `flatMap` operate for those types. For example, the `Monad[Function0]` instance defines `unit[A]` as `() => a` and `flatMap[A,B]` as `() => f(a())()`. Because the `Monad[Function0]` implementation for `flatMap` is not stack-safe, a specialized `translate[F[_],G[_],A]` function is required. This `translate` function uses `runFree` to convert a `Free[F,A]` program into a `Free[G,A]`, preserving the structure but using the translated type $G$ as the base. Once this translation is achieved, a stack-safe `runConsole[A]` function can be implemented to execute the program using the translated structure. Furthermore, the `Free[F,A]` structure's robustness is highlighted by noting that the value does not imply any necessary side effects; the decision to perform I/O is left entirely to the interpreter. This principle allows for multiple interpreters for the same `ConsoleIO[A]` program. For instance, a testing interpreter can be built that translates `Console` requests to `ConsoleReader[A]`, ignoring `PrintLine` requests and always returning a constant string for `ReadLine` requests. Similarly, a second interpreter can translate `Console` requests to `ConsoleState[A]`, which represents a pure state action utilizing a `Buffers` pair (containing an input buffer list and an output buffer list). This `runConsoleState` function allows the program to be tested using a property-based testing library from chapter 8, while `runConsole` runs the program for real. This demonstrates that the core `Free` type is fundamentally independent of side effects, making it a pure, referentially transparent expression that merely makes requests to an interpreter, which is entirely free to use side effects or none.
