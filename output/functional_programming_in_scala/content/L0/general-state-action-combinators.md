# general-state-action-combinators

**Parent:** [[content/L1/functional-state-management-combinators|functional-state-management-combinators]] — Pure functional state management mandates explicit state transitions, using types like `Rand[A]` (defined as `RNG => (A, RNG)`) to track both the resulting value and the next state. Key combinators include `map` (for transforming output state), `map2` (for combining two independent state actions), `sequence` (for listing state actions), and the powerful `flatMap` (for determining the next state action based on the intermediate result).

The initial combinators developed for state management—namely `unit`, `map`, `map2`, and `sequence`—are not exclusively limited to generating random numbers. Instead, they function as general-purpose combinators for handling state actions, abstracting away the specific type of the state being managed. For example, the `map` function does not depend on the state action specifically dealing with a Random Number Generator (RNG) state. Therefore, these functions can be generalized to a common signature that is independent of the state type.

Because `map` and `map2` combinators are insufficient for certain state management tasks, a more powerful combinator, `flatMap`, is required. Developers can implement `flatMap` using the following signature: `def flatMap[A,B](f: Rand[A])(g: A => Rand[B]): Rand[B]`. The `flatMap` combinator allows generating an initial random value of type `A` using `Rand[A]`, and then taking that value `A` to determine which `Rand[B]` action should be executed based on `A`'s specific value. This ability to choose a subsequent state action based on a generated value is crucial for implementing specialized functions like `nonNegativeLessThan`. 

Furthermore, `flatMap` is so powerful that it can be used to reimplement the functionality of `map` and `map2`. Demonstrating that `flatMap` is more powerful than both `map` and `map2` is a key concept in advanced state management.

This generalized state action API can be used to revisit and improve examples, such as the `rollDie` function. Using the pure functional API, the initial implementation of `rollDie` defined as `def rollDie: Rand[Int] = nonNegativeLessThan(6)` can be tested. If a test is performed using a specific initial RNG state, such as `SimpleRNG(5)`, the function might return 0, indicating an error. This bug can be fixed by mapping the result of `nonNegativeLessThan(6)` and adding 1, resulting in the corrected definition: `def rollDie: Rand[Int] = map(nonNegativeLessThan(6))(_ + 1)`.
