# purely-functional-state-management

**Parent:** [[content/L1/functional-programming-principles|functional-programming-principles]] — Functional programming ensures type safety and non-mutation using patterns like Option[+A] and Either[+A, +B] for explicit error handling, and employs techniques like non-strictness (laziness) to achieve automatic loop fusion and optimize data transformations by avoiding intermediate lists. To maintain referential transparency, mutable state, such as random number generation, must be managed by defining an interface (e.g., `RNG`) that forces methods to return both the resulting value and the new state, rather than using side effects.

To recover referential transparency, the state updates must be made explicit. Instead of updating the state as a side effect, a program should return both the newly generated value and the new state. For example, a random number generator should use an interface defined as `trait RNG { def nextInt: (Int, RNG) }`. This method should generate a random `Int` and return both the random number and a new `RNG` object, ensuring the old state remains unmodified. This pattern separates the concern of computing the next state from the concern of communicating the new state to the rest of the program, and no global mutable memory is used; the calling program receives the next state and maintains complete control over what to do with it. Functionally, the process of computing the next state becomes independent of the mechanism used to communicate the new state. The provided implementation, `SimpleRNG`, is a simple random number generator that uses a linear congruential generator algorithm, similar to `scala.util.Random`. The class `SimpleRNG(seed: Long)` extends `RNG` and implements `nextInt: (Int, RNG) = { val newSeed = (seed * 0x5DEECE66DL + 0xBL) & 0xFFFFFFFFFFFFL; val nextRNG = SimpleRNG(newSeed); val n = (newSeed >>> 16).toInt }` to generate the next value. This function returns a tuple containing the pseudo-random integer `n` and the next `RNG` state, `nextRNG`. When using this API in the Scala interpreter, calling `val (n1, rng2) = SimpleRNG(42).nextInt` yields `n1: Int = 16159453` and `rng2: RNG = SimpleRNG(1059025964525)`. Calling the method a second time, `val (n2, rng3) = rng2.nextInt`, yields `n2: Int = -1281479697` and `rng3: RNG = SimpleRNG(197491923327988)`. Since the result of `nextInt` depends only on the input state and not on any external factors, this API is pure. Although using pure functions for state computation results in an efficiency loss because data cannot be mutated in place, this loss can be mitigated by using efficient purely functional data structures. Furthermore, it is possible in some cases to mutate the data in place without compromising referential transparency, a concept to be discussed later.
