**2.5.2 Calling HOFs with Anonymous Functions**

When using higher-order functions (HOFs), it is often convenient to call these functions using anonymous functions or function literals, rather than having to supply an existing named function. For example, a developer can test the `findFirst` function in the Scala Read-Evaluate-Print Loop (REPL) by executing: `findFirst(Array(7, 9, 13), (x: Int) => x == 9)`, which results in `res2: Int = 1`. This example demonstrates new syntax: `Array(7, 9, 13)` is an array literal that constructs a new array containing three integers without needing a `new` keyword. The syntax `(x: Int) => x == 9` is a function literal, also known as an anonymous function. This function defines the comparison logic inline; it takes one argument named `x` of type `Int` and returns a `Boolean` value indicating if `x` is equal to 9. Generally, the arguments to the function are declared to the left of the `=>` arrow, and the function's body is implemented to the right of the arrow. For instance, to write an equality function that takes two integers and checks if they are equal, a developer could write: `(x: Int, y: Int) => x == y`, which results in `res3: (Int, Int) => Boolean = <function2>`. The `<function2>` notation indicates that the resulting value, `res3`, is a function accepting two arguments. If Scala can infer the types of the function’s inputs from the context, the explicit type annotations on the function’s arguments can be omitted, as demonstrated by the simpler form: `(x, y) => x < y`.

**Functions as Values in Scala**

When a developer defines a function literal, Scala actually defines an object that includes a method called `apply`. Scala has a special rule for the `apply` method name, allowing objects that possess this method to be called as if they were methods themselves. For example, defining a function literal like `(a, b) => a < b` is syntactic sugar for object creation: `val lessThan = new Function2[Int, Int, Boolean] { def apply(a: Int, b: Int) = a < b }`. The variable `lessThan` has the type `Function2[Int, Int, Boolean]`, which is conventionally written as `(Int, Int) => Boolean`. Developers should note that the `Function2` interface (known in Scala as a trait) includes an `apply` method. When calling `lessThan` with `lessThan(10, 20)`, it is syntactic sugar for calling the `apply` method: `val b = lessThan.apply(10, 20)`, which yields `b: Boolean = true`. `Function2` is a standard Scala library trait, alongside `Function1` and `Function3`, used to represent function objects that take a specified number of arguments.
