# monad-functor-applicative-interface

**Parent:** [[content/L1/functional-fp-abstraction-architecture|functional-fp-abstraction-architecture]] — Functional programming achieves architectural rigor by establishing a hierarchy of abstractions (Functor, Applicative, Monad), which treat computation as first-class values using primitives like `unit` and `flatMap`. Complex state and effects are managed via specialized types—`Stream` (for lazy evaluation and memoization), `Par` (for non-blocking concurrency via the Actor model), and `Free[F[_], A]` (for separating the description of side effects from their execution via trampolining).

The abstract structure of a Monad (M) is foundational to functional programming, unifying concepts like Parser, Gen, Par, and Option. To achieve this unification, the `Monad` trait defines a minimal set of primitives: `unit` and `flatMap`. These two functions allow all conforming types to gain `map` and `map2` automatically. Specifically, the `Monad` trait definition includes: `def map[A,B](ma: F[A])(f: A => B): F[B] = flatMap(ma)(a => unit(f(a)))` and `def map2[A,B,C](ma: F[A], mb: F[B])(f: (A, B) => C): F[C] = flatMap(ma)(a => map(mb)(b => f(a, b)))`. This principle establishes that all Monads are Functors, but not all Functors are Monads. The `Applicative` trait complements this structure by defining `map2` and `unit` as primitives. While `Monad` is the most general interface, `Applicative` is often simpler and more compositional. The `Monad` trait can be made a subtype of `Applicative` by providing default implementations for `Applicative` methods, confirming that all Monads are also applicative functors. Furthermore, general functional operations like `sequence` (which converts `List[F[A]]` to `F[List[A]]`), `traverse` (which applies a function `A => F[B]` over `List[A]]` to produce `F[List[B]]`), `replicateM` (replicating a computation), and `product` (combining two computations into a pair) can be defined using these core primitives, allowing developers to implement functionality once and apply it universally to any type that supplies a `Monad` instance.
