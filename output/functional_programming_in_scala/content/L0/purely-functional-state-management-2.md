# purely-functional-state-management-2

**Parent:** [[content/L1/functional-state-management-combinators|functional-state-management-combinators]] — Pure functional state management mandates explicit state transitions, using types like `Rand[A]` (defined as `RNG => (A, RNG)`) to track both the resulting value and the next state. Key combinators include `map` (for transforming output state), `map2` (for combining two independent state actions), `sequence` (for listing state actions), and the powerful `flatMap` (for determining the next state action based on the intermediate result).

Purely functional state management involves explicitly making state transitions visible. Instead of allowing functions to mutate an internal state, a functional API should be designed such that methods explicitly transform the current state to produce both a result and the next state. For example, if a class used private fields for state, such as `Foo` containing `FooState s`, the purely functional approach requires defining a trait (like `Foo`) where each method explicitly takes and returns the state: `def bar: (Bar, Foo)` and `def baz: (Int, Foo)`. The calling code becomes responsible for passing the newly computed next state through the rest of the program.

For a random number generator (RNG), this pattern is evident: if a previous RNG state is reused, it will yield the same values. To generate two distinct random integers, the process must pass the updated state. Specifically, the function `randomPair` must use the RNG state returned by the first call to `rng.nextInt` to generate the second integer, as demonstrated by the signature `def randomPair(rng: RNG): ((Int, Int), RNG) = { val (i1, rng2) = rng.nextInt; val (i2, rng3) = rng2.nextInt; ((i1, i2), rng3) }`. This mechanism ensures that the function returns the final state (`rng3`) after generating both random numbers, allowing subsequent code to use the new state.

To streamline the manual state passing, functional programmers define specialized utilities. They introduce a type alias `Rand[A]`, which represents a state action: `type Rand[+A] = RNG => (A, RNG)`. A value of type `Rand[A]` is understood as a program that depends on an initial `RNG` state, uses the `RNG` to generate a value of type `A`, and simultaneously transitions the `RNG` to a new state. Methods, such as `RNG`'s `nextInt`, can be converted into values of this new `Rand[Int]` type by defining `val int: Rand[Int] = _.nextInt`.

To combine these state actions and abstract away the explicit passing of the `RNG` state, combinators are developed. A simple state transition is the unit action, defined as `def unit[A](a: A): Rand[A] = rng => (a, rng)`, which accepts an initial `RNG` state and passes it through without change, returning a constant value `a`. Furthermore, the `map` combinator allows transforming the output of a state action without modifying the state. Since `Rand[A]` is equivalent to a function type `RNG => (A, RNG)`, `map` is implemented as a function composition: `def map[A,B](s: Rand[A])(f: A => B): Rand[B] = rng => { val (a, rng2) = s(rng); (f(a), rng2) }`. An example of this usage is `def nonNegativeEven: Rand[Int] = map(nonNegativeInt)(i => i - i % 2)`, which generates an integer greater than or equal to zero and divisible by two by reusing the function `nonNegativeInt`.

Since `map` is insufficient for combining two independent actions, the `map2` combinator is necessary. This function takes two state actions, `ra` and `rb`, and a combining function `f`, returning a new action that applies `f` to the results of both actions: `def map2[A,B,C](ra: Rand[A], rb: Rand[B])(f: (A, B) => C): Rand[C]`. Using this, a function named `both[A,B]` combines two RNG actions into one that generates pairs: `def both[A,B](ra: Rand[A], rb: Rand[B]): Rand[(A,B)] = map2(ra, rb)((_, _))`. For example, the random values for an (Int, Double) pair can be generated as `val randIntDouble: Rand[(Int, Double)] = both(int, double)`.

More advanced combining functions can be created, such as `both(int, double)` and `both(double, int)` to generate pairs of different types. For generating a list of random integers, the `sequence` combinator can combine a list of state actions into a single state transition: `def sequence[A](fs: List[Rand[A]]): Rand[List[A]]`. This allows for the reimplementation of the `ints` function, which generates `List[Int]`, by using `sequence`.
