# functional-design-parallel-computations-summary

**Parent:** [[content/L1/functional-programming-algebra-design|functional-programming-algebra-design]] — The synthesis details advanced functional programming concepts, including using specialized ADTs like `Option` and `Either` for type-safe error handling; implementing non-blocking concurrency via Actors for primitives like `map2`; achieving memory efficiency through non-strictness and `Stream` types; and generalizing combinators (like `choice` to `flatMap`) to form a coherent mathematical algebra. Furthermore, it introduces property-based testing (PBT) using generators (`Gen`) to define system behavior abstractly.

Upon completing the design of a library for defining parallel and asynchronous computations in a purely functional way, the goal was to illustrate the process of functional design and provide insight into common issues and potential solutions for designing APIs. Chapters 4 through 6 demonstrated the principle of separation of concerns by describing a computation using a data type, `Par`, and relying on a separate interpreter to actually execute the parallel tasks by spawning threads. Readers are encouraged to explore the algebra further by considering: 1) implementing a function with the same signature as `map2` but utilizing `flatMap` and `unit`, and determining how its meaning differs from `map2`; 2) identifying potential laws relating `join` to the other primitives of the algebra; and 3) determining if any parallel computations exist that cannot be expressed using the established algebra, or if any computations cannot even be expressed by adding new primitives to the algebra. Developing the ability to recognize which functions are expressible from an algebra, and recognizing the algebra's limitations, is a critical skill in functional programming. For example, a function like `choice` was initially not obvious to express purely in terms of `map`, `map2`, and `unit`, but later it was observed that `choice` is simply a special case of `flatMap`. Furthermore, the ability to reduce an API to a minimal set of primitive functions is extremely useful because primitive combinators often encapsulate complex logic, allowing developers to reuse the logic rather than duplicating it.
