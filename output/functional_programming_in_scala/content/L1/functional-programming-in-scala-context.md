# functional-programming-in-scala-context

**Parent:** [[content/L2/functional-programming-principles-in-scala|functional-programming-principles-in-scala]] — The book *Functional Programming in Scala* systematically teaches FP principles using Scala as a vehicle, demonstrating how to refactor impure code (like a coffee shop charge system) into pure functional forms that achieve superior testability and reusability by separating core business logic from external effects.

The text provides a comprehensive deep dive into the book *Functional Programming in Scala*, authored by Paul Chiusano and Rúnar Bjarnason, and published by Manning Shelter Island. The materials are licensed to Emre Sevinc via <emre.sevinc@gmail.com>, and online information and ordering can be found at www.manning.com. For further assistance, readers should contact the Special Sales Department at Manning Publications Co., located at 20 Baldwin Road, PO Box 761, Shelter Island, NY 11964, via orders@manning.com. The publication is copyrighted to Manning Publications Co. in 2015, and all rights are reserved. The ISBN for the book is 9781617290657, and it was printed in the United States of America. Structurally, the book is divided into four major parts: Part 1: Introduction to Functional Programming (beginning on page 1); Part 2: Functional Design and Combinator Libraries (starting on page 93); Part 3: Common Structures in Functional Design (starting on page 173); and Part 4: Effects and I/O (starting on page 227).

*Functional Programming in Scala* aims to be a resource detailing the principles and practical application of Functional Programming (FP), which is defined fundamentally as a programming style focused on functions—sometimes referred to as “programming with functions.” While the definition of functions can vary, pure functional programming rigorously restricts functions to behave as binary relations, mapping arguments strictly to results. The critical distinction in pure FP is the demand that functions remain pure. While Scala is classified as an impure functional programming language because it admits both pure and impure functions without syntactic or type distinctions, the authors strongly encourage the utilization of pure functions. Side effects, such as using exceptions, I/O operations, or mutation, are not forbidden, but they can be useful for reasons of interoperability, efficiency, or convenience; however, experts typically deem overusing side effects to be poor programming style.

The authors address potential structural skepticism regarding the best way to learn FP. They note that although some might suggest a detour through a pure functional language like Haskell, this argument is significantly weakened by the emergence of *Functional Programming in Scala*. Consequently, Paul and Rúnar strategically treat Scala as if it were a pure functional programming language, systematically eliminating signs of impurity—specifically, mutable variables, exceptions, classical input/output, and all other impure elements. They prove that every concept can be expressed using only pure functions, demonstrating that this approach yields not only functional possibility but also leads to 'beautiful code' and deep insights into computation.

Paul Chiusano and Rúnar Bjarnason acknowledge that the material is challenging, requiring readers to pay close attention to detail and potentially challenging deeply held programming views. They emphasize that the benefit of the book lies in its self-contained nature: it begins with the simplest expressions and explains every abstraction in granular detail before constructing larger ones. Essentially, the book establishes an 'alternative Scala universe' where mutable state is absent and all functions are pure. Although many commonly used Scala libraries operate by deviating from this ideal (often using partly imperative implementations with a mostly functional interface), the strength of Scala's ability to encapsulate mutable state within a functional interface is acknowledged, though the authors caution against the misuse of this capability, positioning *Functional Programming in Scala* as the primary remedy.

Regarding the genesis of the book, the authors initially encountered difficulty learning FP, recognizing its potent ability to create programs that are coherent, composable, and beautiful. They learned FP piecemeal by writing functional code, engaging with other Scala and Haskell programmers, and gathering information from various fragmented resources. In April 2010, Nermin Šerifović, an organizer within the Boston Area Scala Enthusiasts group, suggested the authors compile a comprehensive text specifically on FP in Scala. After an intensive period spanning over four years of work, they finalized this resource, which they intended to be the definitive guide they had wished existed during their own learning journey.

The book is structured as an introduction to FP—a radical, principled approach to writing software—with Scala serving merely as the vehicle for instruction; the principles taught are applicable across any language. The overall structure, while designed to be read sequentially, is organized with four parts designed to allow for comfortable breaks. Part 4, concerning real-world applications, I/O (e.g., working with databases, files, or video displays), and the utilization of mutable state, is intended to be most comprehensible only after the reader has built a strong foundational familiarity with the functional style introduced in Parts 1, 2, and 3. 

Part 1 covers core concepts, providing an overview of fundamental techniques such as organizing small functional programs, defining purely functional data structures, handling errors, and managing state. Part 2 focuses on tutorials in functional design, illustrating the underlying thought process by working through practical functional libraries. Part 3 elevates the discussion to introduce higher abstractions necessary for writing more generalized libraries, exposing patterns and duplication noticed in Part 2, and presenting powerful tools for reasoning about code that promise exceptional programmer productivity upon mastery. Finally, Part 4 guides the reader across the crucial gap toward writing functional programs that interact with the outside world, utilizing mutable state and performing I/O entirely within a purely functional methodology.

From a pedagogical standpoint, the authors heavily rely on carefully sequenced programming exercises, underscoring that grasping functional programming demands active participation and coding, not just theoretical understanding. Every chapter generally follows a pattern: a new idea or technique is introduced, explained with an example, and then reinforced with a number of exercises. Readers are strongly advised to download the dedicated exercise source code and complete these exercises as they advance through the material. Support for these exercises, including hints and answers, is available at https://github.com/fpinscala/fpinscla. For further discussion and assistance, readers are directed to the scala-functional Google group (https://groups.google.com/forum/#!topic/scala-functional/) and the #fp-in-scala IRC channel on irc.freenode.net. The initial target audience was conceived as the intermediate-level Java or C programmer, yet the authors assert that the material is appropriate for programmers from any language and any professional experience level, emphasizing that curiosity and motivation are more vital than existing expertise. If readers require deeper knowledge of Scala's syntax or features, they are recommended to consult external resources, such as the official Scala language documentation (http://scala-lang.org/documentation/books.html) or specific documentation pages (http://scala-lang.org/documentation/).
