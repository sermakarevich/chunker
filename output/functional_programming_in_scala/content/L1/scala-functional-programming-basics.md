# scala-functional-programming-basics

**Parent:** [[content/L2/functional-programming-principles-in-scala|functional-programming-principles-in-scala]] — The book *Functional Programming in Scala* systematically teaches FP principles using Scala as a vehicle, demonstrating how to refactor impure code (like a coffee shop charge system) into pure functional forms that achieve superior testability and reusability by separating core business logic from external effects.

Scala's functional programming paradigms build upon its object-oriented foundation, treating functions as first-class values. This means that functions can be treated, passed, and assigned to variables just like basic data types such as integers, strings, or lists. Consequently, functions can be used as arguments to other functions, a concept central to Higher-Order Functions (HOFs). The ability to treat functions as values is fundamental to writing purely functional and highly abstract code.

## Children
- [[content/L0/passing-functions-to-functions|passing-functions-to-functions]] — The `formatResult` function is a higher-order function that accepts a name, an integer `n`, and a function `f` of type `Int => Int` to calculate and format the result, as seen in `def formatResult(name: String, n: Int, f: Int => Int) = { val msg = "The %s of %d is %d." msg.format(name, n, f(n)) }`.
- [[content/L0/polymorphic-functions-and-higher-order-functions|polymorphic-functions-and-higher-order-functions]] — The `formatResult` function demonstrates passing functions (`abs` or `factorial`) of type `Int => Int` as arguments, yielding outputs such as "The absolute value of -42 is 42." and "The factorial of 7 is 5040.". Polymorphism, specifically parametric polymorphism, allows developers to generalize functions like `findFirst`, which originally searched for a `String` in an `Array[String]`, to work for any type `A` by accepting a type parameter and a predicate function `p: A => Boolean`.
- [[content/L0/passing-anonymous-functions|passing-anonymous-functions]] — The `findFirst` function requires that the type variable `A` be consistent across its array elements (`Array[A]`) and the predicate function `p` (`A => Boolean`), which the compiler enforces. Programmers can use anonymous functions, such as `(x: Int) => x == 9`, to pass comparison logic to higher-order functions, and function literals are first-class values represented by Scala objects (e.g., `Function2[Int, Int, Boolean]`).
