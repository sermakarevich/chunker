# claude-opus-4-7-safety-evaluations

**Parent:** [[content/L2/anthropic-model-evaluations-benchmarks|anthropic-model-evaluations-benchmarks]] — Anthropic's evaluations revealed that Claude Opus 4.7 shows strong performance improvements, achieving a 99.92% harmless rate in single-turn child safety testing and demonstrating superior robustness in malicious code refusal (91.15% in Claude Code), but it consistently underperforms Claude Mythos Preview in key areas, such as general capability assessment, CyberGym, and the CB-2 biological sequence design task.

The release of Claude Opus 4.7 was accompanied by extensive agentic safety evaluations, building upon methodologies established with predecessors like Claude Mythos Preview. Anthropic introduced two new, advanced benchmarks to address the saturation of existing security testing suite: one specifically assessing the model's capability to autonomously execute influence operation campaigns, and another designed to test resistance against indirect prompt injection. Furthermore, the evaluation framework streamlined certain tests; specifically, an older evaluation concerning basic malicious coding tasks, detailed in Section 5.1.1 of the Claude Opus 4.6 System Card, was removed because it had reached a highly saturated state. This functionality was replaced by more challenging evaluations, particularly those focused on Claude Code and general computer use tasks.

### Malicious Use and Coding Environments

**Claude Code Evaluation:** This dedicated evaluation assesses how the model responds to diverse cyber-related queries, spanning malicious, dual-use, and benign scenarios, all conducted within the specialized context of Claude Code. The malicious prompts used in this assessment cover 61 distinct topics, rigorously testing Claude’s ability to correctly decline any query that exhibits malicious intent or contravenes the established Usage Policy. Illustrative malicious topics include providing assistance with malware creation, drafting code for destructive DDoS attacks, and developing software for non-consensual monitoring. Since the initial Claude Mythos Preview System Card, Anthropic significantly augmented the number of test cases, adding 12 challenging cases, thereby raising the total number of malicious prompts from 49 to 61. Complementing the malicious set, a comprehensive collection of 61 dual-use and benign queries was utilized. These non-prohibited queries evaluate Claude’s capacity to assist with requests that, while potentially sensitive, do not inherently violate usage policies. Examples of dual-use/benign topics include running network reconnaissance tools, testing websites for potential vulnerabilities, and conducting detailed analysis on data sourced from a penetration test. For the Opus 4.7 evaluation, the model was tested using the standard set of tool commands available in Claude Code, with every single prompt executed 10 times, resulting in a massive total of 1,220 tests spanning both malicious and dual-use/benign categories. Additionally, Anthropic now automatically executes this entire evaluation using the Claude Code system prompt, which anticipates its default implementation across all subsequent model releases. A previous safeguard involved a FileRead tool reminder that explicitly instructed the model to consider the potential maliciousness of a file. However, evaluations conducted for recent models, including Opus 4.7 and Mythos Preview, demonstrated that this specific mitigation did not furnish any measurable additional security benefit. Consequently, Anthropic now applies this safeguard only when it yields a material security improvement, ensuring that the reported score is provided both with and without the FileTool reminder. System prompt updates were also implemented, subtly altering the results for previous model generations.

**Opus 4.7 Performance in Code:** The results of the malicious use evaluation highlight that Opus 4.7 demonstrated a substantial improvement compared to its predecessor, Claude Opus 4.6, particularly in recognizing and refusing malicious requests. This improvement mirrored the trends observed in Claude Mythos Preview, with the enhanced performance on Opus 4.7 originating from successfully handling challenging new cases where prior models had shown difficulty. Conversely, regarding dual-use and benign tasks, Opus 4.7 performed comparably to Claude Opus 4.6. The measurable performance metrics for this evaluation are detailed below:

**Computer Use Simulations:** Anthropic executed the computer use evaluation for Claude Opus 4.7 using the version of the model last updated for the launch of Claude Opus 4.5. This evaluation meticulously assesses how the model responds to harmful tasks when it is provided with GUI- and CLI-based computer use tools within a fully contained, sandboxed environment. The focus was placed on three specific high-risk areas: 1) unauthorized data collection and surveillance, 2) the generation and distribution of harmful content, and 3) scaled abuse potential. A total of 112 tasks were executed, with the evaluation run 224 times—once with thinking enabled and once with thinking disabled—for each model.

**Malicious Computer Use Results:** The refusal rates showed Opus 4.7 scoring higher than Claude Opus 4.6, though this improvement was within the standard margin of error of other recently evaluated models. The comparative data was:

### Advanced Agentic and Indirect Attack Vectors

**Malicious Agentic Influence Campaigns:** This new benchmark, first mentioned in the system card for Claude Mythos Preview, is designed to evaluate potential risks of harmful manipulation on a large scale. The test assesses whether the model has the inherent capacity to autonomously conduct an influence operation at a level that would provide a meaningful uplift to a malicious actor. Such an influence operation is formally defined as a coordinated campaign intended to shape public opinion, influence political behavior, or distort public discourse through deception. This can involve methods such as utilizing inauthentic personas, creating fabricated support for grassroots movements, or generating targeted disinformation for distribution across social and digital media platforms. The primary concern addressed by this evaluation is scale: tasks that previously required dedicated teams of human operatives—including drafting content consistent with a persona, adapting messaging for various target audiences, and maintaining sustained engagement across numerous accounts—could potentially be automated end-to-end by the model. The evaluation is structured to measure the model’s capacity to execute a complete campaign end-to-end, specifically designed to circumvent existing platform friction and counter-engagement defenses. This assessment takes place within an agentic harness, granting the model access to simulated social media platform tools within a mocked ecosystem that includes integrated moderation mechanisms and counter-engagement obstacles.

**Indirect Prompt Injection (ART Benchmark):** The Agent Red Teaming (ART) benchmark, a collaboration with the UK AI Security Institute and evaluated by external research partner Ray Swan, assesses a model's vulnerability to prompt injection across four critical categories: breaching confidentiality, introducing competing objectives, generating prohibited content (such as malicious code), and executing prohibited actions (like unauthorized financial transactions). The section specifically focuses on indirect prompt injection, which refers to malicious instructions that are embedded within external data. Ray Swan established that attacks are not deterministic, and therefore, he estimated the probability of an adversary succeeding by setting three variables for the number of attempts: $k=1$, $k=10$, and $k=100$. The attacks utilized are drawn from the ART Arena, where thousands of expert red teamers continuously refine their strategies against cutting-edge models. For this analysis, Swan selected a defined subset of attacks known to be effective across multiple models, rather than focusing solely on the model originally targeted. The results shown in Figure 5.2.1.A represent the likelihood that an attacker successfully penetrates the system after 1, 10, and 100 attempts, with a lower success rate indicating stronger model robustness.

**Shade Tool Evaluations:** For assessing coding environments, the Shade tool, an external adaptive red-teaming utility developed by Gray Swan, evaluates model robustness against prompt injection attacks within coding environments. Shade agents are sophisticated tools that integrate search, reinforcement learning, and human-in-the-loop insights to continuously improve their ability to exploit model vulnerabilities. The strong variant of this evaluation applies heightened adversarial pressure on the tested models. The reporting structure measures the percentage of test cases where the attacker succeeds after 1 and 200 attempts. These comparisons are conducted both with and without additional user-protective safeguards.

*   **Coding Environment Results:** When assessing coding environments, Opus 4.7 demonstrated an improvement in robustness when compared to both Claude Opus 4.6 and Claude Sonnet 4.6. *Without safeguards*, Claude Opus 4.7 significantly reduced the attack success rate (ASR) for 1 attempt from 25.92% (Opus 4.6) down to 2.34%. At 200 attempts, the rate dropped from 97.5% (Opus 4.6) to 60.0% with adaptive thinking. When adaptive thinking was disabled, the rates saw reductions from 54.14% (Opus 4.6) to 10.43% at 1 attempt, and from 100% (Opus 4.6) to 92.5% at 200 attempts. Critically, applying additional safeguards further decreased these rates to 0.43% at 1 attempt and 25.0% at 200 attempts.

*   **Computer Use Environment Results:** Using the Shade adaptive attacker to evaluate robustness in computer use environments, where the model interacts directly with the GUI, Opus 4.7 also exhibited improvements over Claude Opus 4.6. *Without safeguards*, Opus 4.7 showed reduced ASR with adaptive thinking (from 78.6% for Opus 4.6 to 0.46% at 1 attempt, and from 50.0% to 7.14% at 200 attempts) and without thinking (reduced from 20.0% for Opus 4.6 to 0.39% at 1 attempt, and from 85.7% to 21.43% at 200 attempts). Interestingly, adding safeguards *increased* the attack success rates for Claude Opus 4.7 in this specific evaluation across both adaptive thinking and no thinking scenarios. The developers noted that this increase was not deemed statistically significant due to the low overall attack success rates and the small number of test cases. The analysis cautioned that all reported attack success rates are based on a strong adversary optimized specifically against Claude with attacks that the model always encounters, meaning real-world deployment attack success rates would likely be even lower.

**Browser Use Robustness:** An internally developed evaluation measures browser use robustness, primarily intended for products like the Claude in Chrome extension and the Claude Cowork tool. This evaluation was launched with Claude Opus 4.5. Because successive models have saturated earlier attack sets, the evaluation has been periodically refreshed with more complex environments and increasingly strong attacks. The method involves utilizing web environments where the team dynamically injects untrusted content into pages that the model later views, either through page reads or screenshots. The report quantifies the Attack Success Rate (ASR) as the fraction of injected content that succeeded out of the content the model actually viewed, acknowledging that model navigation patterns can cause the model to miss some injected content. Success is verified via a programmatic checker within the environment. Comparative analysis was performed across 148 total environments (with 10 attempts each). The table summary details the attack rate for four models under varying conditions:

| Model | Condition | ASR (Opus 4.7) | ASR (Opus 4.6) |
| :--- | :--- | :--- | :--- |
| Opus 4.7 | With/Without Thinking | 54.05% | 80.41% (With Thinking) |
| Opus 4.7 | With Safeguards | 0.75% (Without Thinking) | 2.70% (Opus 4.6) |

*   **Performance Comparison:** In the 'Without safeguards' column, Claude Opus 4.7 achieved a per-environment ASR of 54.05% regardless of thinking status. In contrast, Claude Opus 4.6 reached significantly higher rates, achieving 86.49% without thinking and 80.41% with thinking. The analysis provided a critical caveat that the sourced attacks were adaptively generated against Claude Opus 4.6, requiring cautious interpretation of the observed differences between the two models. When safeguards were deployed, Claude Opus 4.7 demonstrated a markedly reduced per-environment ASR of 0.75% (without thinking) and 0.74% (with thinking). This represents the strongest performance recorded on this benchmark, matching the results achieved by Claude Mythos Preview. For the older generation models, safeguards reduced the attack success rates for Claude Sonnet 4.6 and Claude Opus 4.6 to approximately 2.70% both with and without thinking.

**General Misalignment Assessment:** Finally, the report includes an assessment of general misalignment, focusing on Claude Opus 4.7's behavioral integrity. This analysis investigates concerning behaviors such as displaying undesirable or hidden goals, knowingly assisting misuse, utilizing reasoning scratchpads in deceptive or unreliable ways, exhibiting sycophancy toward users, making attempts to undermine safeguards, hiding dangerous capabilities, or attempting to manipulate users toward predetermined views. This investigation was exhaustive, drawing from multiple methodologies, including static behavioral evaluations, automated interactive behavioral evaluations, dictionary-learning interpretability methods, white-box steering and probing methods, non-assistant persona sampling, 14 specific misalignment-related capability evaluations, thorough training data review, and extensive feedback from both internal and external pilot users. The process involved the manual expert inspection of thousands of transcripts and the automatic screening of reinforcement-learning training transcripts, consuming over a hundred hours of expert time. In terms of overall safety and alignment, the assessment found that Opus 4.7's capabilities are broadly similar to those of Claude Opus 4.6 and Claude Sonnet 4.6 on core measures of potential for high-stakes misuse. Specifically, Opus 4.7 displayed significant improvements over Claude Opus 4.6 and Claude Sonnet 4.6 when handling misuse within the constrained environments of Claude Code and GUI computer-use sessions. Nevertheless, the model exhibited modest regressions relative to Claude Opus 4.6 in certain other areas of misuse, and it remained noticeably weaker than Claude Mythos Preview. The overall assessment confirmed that while Opus 4.7's rate of over-refusals is lower than most prior models and comparable to Opus 4.6 and Mythos Preview, the model was deemed largely well-aligned and trustworthy, though not fully ideal in its behavior.
