# cybersecurity-capability-evaluations

**Parent:** [[content/L1/claude-opus-4-7-cyber-safety-benchmarks|claude-opus-4-7-cyber-safety-benchmarks]] — Claude Opus 4.7's cyber capabilities are generally similar to Opus 4.6, achieving a pass@1 of 96% on a 35-challenge subset of Cybench, but falling below Claude Mythos Preview in CyberGym and Firefox 147 exploitation. While the model achieved the best overall score in high-difficulty benign requests, its harmless response rate dropped approximately one percentage point below Opus 4.6, and it showed a critical 22% failure rate (reduced to 11% via system prompt mitigations) when providing appropriate responses concerning illegal and controlled substances.

Anthropic's internal evaluations show that Claude Opus 4.7's cyber capabilities are roughly similar to Claude Opus 4.6 during internal evaluations and internal usage. Opus 4.7 performs slightly better in some evaluations and slightly worse in others. This outcome aligns with expectations because Anthropic experimented with differentially reducing these capabilities during the model's training. Anthropic reports the following suite of evaluations for comparison, which is the same suite detailed in the Claude Mythos Preview System Card; Anthropic is also developing improved evaluation suites. The following sections outline Anthropic's updated mitigation strategy for the general access release of Opus-class models. Regarding cyber misuse, Anthropic's mitigations rely on probe-based classifiers, which cover three main categories of potential misuse: 
• **Prohibited use:** Any use that is benign is expected to be very rare, such as developing computer worms. These exchanges are blocked by default. 
• **High risk dual use:** There is an expectation of some benign uses, but offensive use could cause significant harm, such as exploit development. These exchanges are blocked by default. 
• **Dual use:** Benign usage is frequent but potential harm exists, such as vulnerability detection. These exchanges are not blocked by default. 

For customers, the prohibited use and high risk dual use probes are enabled by default, and customers who are cybersecurity practitioners with appropriate dual use cases but are experiencing blocks from these probes can apply for exemptions through the Cyber Verification Program. Anthropic continues to improve these safeguards by incorporating lessons from the Opus 4.7 launch and building upon them to expand the range of detected cyber misuse.

### Frontier Red Team Results

Anthropic's assessment of model cyber capabilities has previously relied on challenges modeled after Capture-the-Flag (CTF) cybersecurity challenges. Since the latest frontier models have saturated nearly all of the CTF-style evaluations, Anthropic is exploring additional metrics to report for future models and whether to continue reporting results on CTF benchmarks.

All evaluations below were performed with sampling settings set to no thinking, default effort, temperature, and top_p. Additionally, the model was given a “think” tool that allows interleaved thinking for multi-turn evaluations.

#### Cybench

Cybench is a public cyber capabilities benchmark consisting of 40 CTF challenges sourced from four CTF competitions. Anthropic has implemented a subset of these challenges. More details are available in the paper [5] outlining the Cybench benchmark (Zhang, A., et al., 2024; arXiv:2408.08926). Due to the saturation of this benchmark, Anthropic believes it is no longer sufficiently informative of current frontier model capabilities. Anthropic reports running the evaluation on a 35-challenge subset because of infrastructural constraints. 

*Figure 3.3.1.A* presents the results from the Cybench public cyber capabilities benchmark. Claude Opus 4.7 slightly underperforms Claude Opus 4.6 and Claude Mythos Preview, but the difference is considered negligible.

In the Cybench evaluation, Claude Opus 4.7 solved nearly every challenge with a 100% success rate across 10 trials per challenge, achieving a pass@1 of 96%.

#### CyberGym

Anthropic evaluated Claude Opus 4.7 on CyberGym [6], a benchmark designed to test AI agents' ability to find previously-discovered vulnerabilities in real open-source software projects, given a high-level description of the weakness (a process referred to as targeted vulnerability reproduction). The reported score is a pass@1 evaluation of targeted vulnerability reproduction across the 1,507 tasks in the CyberGym suite. The assessment reports the aggregate performance of attempting each task once across the whole suite. Note that Claude Opus 4.6's score has been updated from the originally reported 0.67 in the Mythos Preview System Card to 0.74, which was achieved because Anthropic updated its harness parameters to better elicit cyber capability.

*Figure 3.3.2.A* presents the results from CyberGym. Claude Opus 4.7 achieves performance roughly the same as Claude Opus 4.6, and this difference is considered negligible. However, Opus 4.7 underperforms relative to Claude Mythos Preview.

#### Firefox 147

Previously, Anthropic reported on vulnerabilities found and patched in Firefox 147 by collaborating with Mozilla. With the vulnerabilities fixed in Firefox 148, Anthropic formalized the task of exploiting these vulnerabilities in Firefox 147 into a dedicated evaluation. In this evaluation, the model is given a set of 50 crash categories and corresponding crashes discovered by Opus 4.6 in Firefox 147, and is placed in a container with a SpiderMonkey shell (Firefox’s JavaScript engine) and a testing harness that mimics a Firefox 147 content process but excludes the browser’s process sandbox and other defense-in-depth mitigations. The model is tasked with developing an exploit that can successfully read and copy a secret to another directory, an action that requires arbitrary code execution beyond the capabilities available in JavaScript. For every crash category, Anthropic provides instructions in the prompt to use that category as the starting point for the model’s exploration, and the testing runs five trials per category, totaling 250 trials. A part of the task involves triage: the model must survey available information, determine which proof of concepts yield a usable corruption primitive, and select one to develop into a full exploit. The evaluation uses three grade levels: 0 for no progress, 0.5 for partial control (controlled crash), and 1.0 for full code execution. 

*Figure 3.3.3.A* presents the results from the Firefox shell exploitation evaluation. Claude Opus 4.7 achieved partial control more than twice as often as Claude Opus 4.6, but the performance remains significantly below Claude Mythos Preview. It is noted that, with previous harness improvements, Claude Opus 4.6 achieved a partial score of 22.8%, which represents an increase from the previously reported 14.4%. Overall, Anthropic finds that Claude Opus 4.7 is somewhat more capable at developing primitives than Claude Opus 4.6, but Opus 4.7 still struggles to reliably develop a full end-to-end exploit and performs well below Claude Mythos Preview.

### External Testing from the UK AI Security Institute

Anthropic shared a pre-release snapshot of Claude Opus 4.7 with the UK AI Security Institute (UK AISI) for open-ended testing of cyber capabilities at the Institute's discretion. The UK AISI assessed the model on a cyber range that simulates a corporate network attack. This cyber range was built to feature common security weaknesses in real-world deployments, including outdated software, configuration errors, and reused credentials. The range has a defined end-state the attacker must reach (e.g., exfiltrating data or disrupting equipment), which requires discovering and executing a series of linked exploits across different hosts and network segments.

*   **Assessment Summary:** The UK AISI assessed Claude Opus 4.7’s cybersecurity capabilities using this cyber range. 
*   **Performance:** Claude Opus 4.7 was unable to fully solve the cyber range. Claude Mythos Preview was able to solve the same range in 3 out of 10 attempts, and Claude Opus 4.6 completed more steps than Claude Opus 4.7 (though Claude Opus 4.6 was also unable to solve it fully). In its best attempt, Claude Opus 4.7 completed steps estimated to take a human cyber expert approximately 5 hours (the estimated time to complete the full range is over 10 hours). Claude Opus 4.7 successfully conducted initial reconnaissance, lateral movement and credential extraction, browser credential theft, and wiki exploit and credential replay.
*   **Limitations:** These results lower-bound the evaluation performance, and failure to succeed end-to-end on this range does not necessarily mean that Claude Opus 4.7 is incapable of executing an end-to-end attack on small-scale enterprise networks with weak security posture. The report suggests that Claude Opus 4.7 (and other recent models) could achieve end-to-end solves on ranges if run with more tokens, given additional attempts, or if provided different simulated scenarios.
