# deep-research-benchmarks

**Parent:** [[content/L1/opus-4-7-multi-benchmark-evaluation|opus-4-7-multi-benchmark-evaluation]] — Claude Opus 4.7 demonstrated strong performance across varied benchmarks, achieving an 87.6% score on the SWE-bench Verified subset and an 86.3% score on OfficeQA, and a 79.3% score on the BrowseComp web search benchmark. In agentic and coding security tests, Opus 4.7 showed significant gains, dropping the Attack Success Rate in the Coding Environment from 25.92% (for Opus 4.6) to 2.34% at 1 attempt without safeguards.

When evaluating agent capabilities, the **BrowseComp** benchmark tests an agent’s ability to find hard-to-locate information on the open web. When running Claude Opus 4.7 with web search, web fetch, programmatic tool calling, and code execution, Opus 4.7 scored 79.3% with thinking off at max effort and a 10M token limit. Using context compaction (triggered at 200k tokens) allowed the evaluation to extend beyond the 1M context window. The analysis found that adaptive thinking performed slightly worse for Opus 4.7 on this benchmark. In comparison, Claude Opus 4.6 demonstrated a better test-time compute scaling curve than Opus 4.7 and achieved a better score on BrowseComp, scoring 83.7% compared to Opus 4.7's 79.3% at the 10M token limit. However, the authors recommend users experiment with both Claude Opus 4.7 and Claude Opus 4.6 because Opus 4.7 is generally more token-efficient than Claude Opus 4.6 on search benchmarks.

For deep research, the **DeepSearchQA** benchmark is a 900-prompt task designed to evaluate agents on difficult multi-step information-seeking tasks across 17 different fields, requiring extensive searches to compile comprehensive answers. For this test, the Claude models were run using web search, web fetch, programmatic tool calling, context compaction, max reasoning effort, and adaptive thinking. For Claude Mythos Preview and Claude Opus 4.7, context compaction was triggered at 200k tokens, while Claude Opus 4.6 used a trigger of 50k tokens. Using the DeepSearchQA results, Claude Opus 4.7 achieved an F1 score of 9.1% (8 ± 1.8% for fully correct answers, 0.7% ± 2.6% for fully incorrect answers, and 7.0% ± 1.7% for correct answers with excessive answers). Claude Opus 4.6 achieved an F1 score of 1.3% (9 ± 1.6% for fully correct answers, 0.6% ± 2.6% for fully incorrect answers, and 0.0% ± 1.4% for correct answers with excessive answers). Claude Sonnet 4.6 achieved an F1 score of 0.5% (9.8% ± 2.6% for fully correct answers, 0.1% ± 1.4% for fully incorrect answers, and 0.9% ± 1.5% for correct answers with excessive answers).

Additionally, the **Deep Research Accuracy, Completeness, and Objectivity (DRACO)** benchmark, developed by Perplexity, evaluates deep research questions that real users would ask. DRACO consists of 100 curated tasks across various domains, which are graded using expert rubrics covering factual accuracy, breadth and depth of analysis, presentation quality, and citation quality. When evaluating Claude models with web search, web fetch, programmatic tool calling, and code execution, Claude Opus 4.7 scored 77.7% using adaptive thinking at max effort and a 1M token limit. This evaluation used Claude Opus 4.6 as the LLM judge, grading responses against per-task rubrics, and reporting the mean score from 5 independent grading runs per response. Opus 4.7's improvement over Opus 4.6 was notable in breadth & depth of analysis and citation quality, while factual accuracy and presentation remained roughly unchanged.

Finally, in the **AB-Bench FigQA** benchmark, a visual reasoning test part of the Language Agent Biology Benchmark (LAB-Bench) developed by FutureHouse, Opus 4.7 demonstrated performance improvements based on image resolution. With adaptive thinking, max effort, and no tools, Claude Opus 4.7 achieved a score of 78.6% on FigQA. When Python tools were available, Opus 4.7 achieved an even higher score of 86.4%. The analysis showed a significant uplift in performance when increasing the maximum image resolution Opus 4.7 could process to 2576px along a single dimension and 3.75MP total, compared to the previous maximum resolution of 1568px along a single axis and 1.15MP total. With Python tools, this improvement raised the score from 85.8% to 86.4% for Opus 4.7.
