# benchmarking-scores

**Parent:** [[content/L1/opus-4-7-multi-benchmark-evaluation|opus-4-7-multi-benchmark-evaluation]] — Claude Opus 4.7 demonstrated strong performance across varied benchmarks, achieving an 87.6% score on the SWE-bench Verified subset and an 86.3% score on OfficeQA, and a 79.3% score on the BrowseComp web search benchmark. In agentic and coding security tests, Opus 4.7 showed significant gains, dropping the Attack Success Rate in the Coding Environment from 25.92% (for Opus 4.6) to 2.34% at 1 attempt without safeguards.

The evaluation utilizes several specialized benchmarks to assess model capabilities. For the Software Engineering Bench (SWE-bench), four variants were tested: **SWE-bench Verified** (a 500-problem subset verified by human engineers, achieving an 87.6% score for Claude Opus 4.7), **SWE-bench Pro** (a harder variant using problems from actively-maintained repositories with larger, multi-file diffs and no public ground-truth leakage, where Opus 4.7 achieved 64.3%), **SWE-bench Multilingual** (extending the format to 300 problems across 9 programming languages, scoring 80.5% for Opus 4.7), and **SWE-bench Multimodal** (adding visual context like screenshots and design mockups to issue descriptions, where Opus 4.7 achieved 34.5% on an internal harness).

For software engineering tasks, the **Terminal-Bench 2.0** was used, a test developed by researchers at Stanford University and the Laude Institute that evaluates AI models on real-world command-line tasks. The evaluation ran the tasks in the Harbor scaffold using the Terminus-2 harness and default parser. Claude Opus 4.7 achieved a mean reward of 69.4%, averaged over 5 attempts for all 89 unique tasks (totaling 445 trials), utilizing the Terminus-2 harness with thinking disabled. The score is calculated using a production API endpoint to account for inference latency dynamics.

Regarding knowledge benchmarks, the **Graduate-Level Google-Proof Q&A (GPQA)** benchmark, which features 198 challenging multiple-choice science questions (using the Diamond subset), yielded a score of 94.2% for Claude Opus 4.7, averaged over 10 trials. For general knowledge and reasoning, the **Multilingual Massive Multitask Language Understanding (MMLU)** benchmark tested knowledge across 57 academic subjects in 14 non-English languages. Claude Opus 4.7 achieved 91.5% on all non-English language pairings, averaging over 3 trials, using adaptive thinking, max effort, and default sampling settings. The **USA Mathematical Olympiad (USAMO)**, a six-problem, two-day proof-based competition for high school students, was assessed using the MathArena grading methodology. Claude Opus 4.7 scored 69.3%, averaged over 10 attempts per problem, using medium effort in the batch API with a 300k token limit.

In long context evaluation, the **GraphWalks** benchmark is a multi-hop test where the context window is filled with a directed graph of hexadecimal-hash nodes; Opus 4.7 scored 58.6% on BFS 256K-1M and 75.1% on parents 256k-1M, averaged over 5 trials. When utilizing the **OpenAI Multi-Round Co-Reference Resolution (MRCR) v2** benchmark, the model's performance measured the ability to locate and distinguish between multiple similar pieces of information in long contexts. For the reported variants, Opus 4.7's scores are reported as the Mean Match Ratio using the 8-needle variants, where 256k bins represent prompts with (128k, 256k] tokens and 1M bins represent prompts with (524k, 1024k] tokens.

Other agentic tasks were assessed using the **Humanity’s Last Exam (HLE)** benchmark, which is a multi-modal benchmark of human knowledge comprising 2,500 questions. For Opus 4.7, the score was 46.9% without tools and 54.7% with tools, both at max reasoning effort. These results were obtained by setting Claude Opus 4.6 as the model grader. Furthermore, the evaluation of **Autonomous Code Generation (ACG)** showed that Opus 4.7 reduced the attack success rate from 25.92% (for Opus 4.6) to 2.34% at 1 attempt without safeguards, and from 97.5% (for Opus 4.6) to 60.0% at 200 attempts with adaptive thinking, when tested in the Shade tool. Similarly, in the GUI computer use evaluation, Opus 4.7 showed improvements, reducing the attack success rate with adaptive thinking from 78.6% (for Opus 4.6) to 0.46% at 1 attempt, and from 50.0% (for Opus 4.6) to 7.14% at 200 attempts without safeguards. The analysis confirms that these reported attack success rates are based on a strong adversary optimized against Claude with attacks always encountered by the model, meaning real-world deployment success rates would likely be lower.
