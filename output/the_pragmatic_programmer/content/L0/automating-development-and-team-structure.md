# automating-development-and-team-structure

**Parent:** [[content/L1/software-development-methodology-synthesis|software-development-methodology-synthesis]] — Effective software development mandates full automation of build, test, and deployment procedures (CI/CD), utilizing Version Control to trigger processes via commits and ensuring roles are not siloed through the 'tracer bullet' method. Teams must adopt systematic estimation techniques like PERT (optimistic/most likely/pessimistic figures) and Iterative Estimation, focusing on delivering working software on demand while resisting the 'cargo cult' trap.

To ensure both consistency and accuracy in software development, the team should automate all processes. The original team's editor or IDE should handle code formatting automatically, and the continuous build should automatically run tests instead of requiring manual testing. Furthermore, automation should manage deployment so that the process is always the same, repeatable, and reliable. Automation is an essential component for every project team, so the team must acquire skills in tool building to construct and deploy the tools necessary to automate project development and production deployment. Regarding team structure, it is crucial to remember that teams are composed of individuals. The team must give each member the ability to shine in their own unique way, providing only enough structure to support the individuals while still ensuring that the project delivers value. The team should resist the temptation to add excess scope or unnecessary complexity, drawing inspiration from the principle described in the 'Good-Enough Software' discussion.
