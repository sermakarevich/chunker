## Top-Level Summaries
- [[content/L1/claude-code-agent-architecture|claude-code-agent-architecture]] — Claude Code, an agentic coding tool, is designed around five core values (Authority, Safety, Reliability, Capability Amplification, Contextual Adaptability), which guide the implementation of architectural principles like deny-first evaluation and five-layer context compaction. It operates a core while-loop loop and relies on complex surrounding subsystems including a seven-mode permission system, four extensibility mechanisms, and an append-oriented session store.
- [[content/L1/agent-architecture-and-context-management|agent-architecture-and-context-management]] — Claude Code implements a highly structured, layered architecture comprising five subsystems (surface, core, safety/action, state, backend). Its context management uses five mandatory shapers—Budget reduction, Snip, Microcompact, Context collapse, and Auto-compact—executed sequentially before every model call. Safety relies on a deny-first rule evaluation passing through seven independent gates, including sandboxing and hook-based interception, thereby ensuring resilience regardless of user vigilance.
- [[content/L1/agent-architecture-mechanisms|agent-architecture-mechanisms]] — The agent's highly layered architecture employs a 'deny-first' authorization pipeline, managed through three mechanisms (declarative rules, global trust modes, and hooks), and five permission modes (e.g., `plan`, `acceptEdits`, `auto`) to enforce safety; context assembly is non-static, starting with a four-level CLAUDE.md memory hierarchy and evolving during the turn through late injections; and extensibility is achieved via four core mechanisms (MCP servers, Plugins, Skills, Hooks) plus agent spawning, which collectively manage context state and tool reach through sophisticated meta-tools and a five-layer compaction process.
- [[content/L1/agent-architecture-comparison|agent-architecture-comparison]] — Claude Code delegates to isolated subagents via AgentTool with six built-in types (Explore, Plan, etc.), while OpenClaw supports advanced multi-agent routing and customizable sub-agent delegation with up to 5 nesting depths, managing security through differing trust models (deny-first vs. single trusted operator).
- [[content/L1/claude-code-operational-harness-design|claude-code-operational-harness-design]] — Claude Code structures its architecture around an operational harness (98.4% of code) that manages state and enforces permissions, allowing the LLM to remain a stateless completion endpoint. Safety is ensured via a multi-stage pipeline checked by the Policy Core, which handles contexts including Coordinator, Swarm worker, Speculative classifier, and Interactive. Context management uses a five-layer compaction pipeline (Budget reduction, Snip, Microcompact, Context collapse, Auto-compact), and subagents utilize the `AgentTool` meta-tool with three isolation modes: Worktree, Remote, and In-process.
- [[content/L1/claude-code-operational-architecture-2|claude-code-operational-architecture-2]] — Claude Code utilizes a highly complex, multi-layered operational harness (98.4% of code) structured around three commitments: graduated layering, append-only auditability, and model judgment within a deterministic framework; it manages safety via a seven-stage pipeline across seven permission modes (e.g., `acceptEdits`, `auto`) and context memory via a five-stage compaction pipeline.
