# claude-code-architecture-harness

**Parent:** [[index]]

Claude Code, an agentic coding tool developed by Anthropic, was designed with an architecture that prioritizes investment in a rich operational harness over limiting the Large Language Model's inherent decision-making processes. The core agent loop functions as the operational harness, while the Large Language Model (LLM) is utilized solely as a stateless completion endpoint. Source code analysis estimates that approximately 98.4% of Claude Code's codebase constitutes the operational harness, leaving only about 1.6% for actual decision logic. This structural separation demonstrates that the architectural complexity is intended not to constrain the model’s decisions, but rather to enable them, adhering to the 'Values over Rules' principle, which stipulates that the operational harness must create conditions that allow the model to make informed decisions rather than dictating those choices. 

To ensure system reliability and control, Claude Code employs a layered safety architecture. Every tool invocation must pass through a multi-stage authorization pipeline, which operates independently of human vigilance. This pipeline consists of three primary components: declarative rules, global trust modes, and programmable hooks. The decision process begins with a Policy Core assessment that determines the outcome as Deny, Allow, or Ask. If the model requests a tool, the process first executes tool pre-filtering, which strips blanket-denied tools from the model's view before tool pool assembly. Next, the PreToolUse hook fires; this hook can return a `permissionDecision` (deny or ask), a `permissionDecisionReason`, or an `updatedInput` to modify tool parameters. Following this, the deny-first rule engine evaluates the request, ensuring that any deny rule always takes precedence over an allow rule. Finally, the Permission handler checks the runtime context, which can be **Coordinator** (for multi-agent coordination), **Swarm worker** (for worker agents), **Speculative classifier** (when `BASH_CLASSIFIER` is enabled and the tool is a `BashTool`), or **Interactive** (the standard user path).

Claude Code supports an extensible architecture that includes four distinct mechanisms: Mechanical Composition Protocol (MCP) servers, plugins, skills, and hooks. The system organizes these extensions by increasing context cost: hooks consume zero context, skills consume low context, and MCP servers consume high context. The process of context management utilizes a five-layer compaction pipeline, necessitated by the context window being the primary binding resource constraint. These five sequential shapers are: 
1. **Budget reduction:** This layer enforces per-tool-result size limits and is always active. 
2. **Snip:** A lightweight trim (gated by `HISTORY_SNIP`) that removes older history segments. 
3. **Microcompact:** A fine-grained, cache-aware compression (gated by `CACHED_MICROCOMPACT`). 
4. **Context collapse:** Performs a read-time projection over the conversation history (gated by `CONTEXT_COLLAPSE`). 
5. **Auto-compact:** Performs a full, model-generated summary (enabled by default). 

For complex workflows, Claude Code manages subagent delegation and orchestration. Subagents, which include built-in types (e.g., Explore, Plan, General-purpose, Claude Code Guide, Verification, Statusline-setup) and user-defined types (using `.claude/agents/*.md` files), operate within isolated contexts. The delegation mechanism uses the `AgentTool` meta-tool, requiring structured input that defines the delegation prompt, an optional subagent type, and specific configurations for isolation mode, permission overrides, and the working directory. The system supports three isolation modes: **Worktree** (creates a temporary git worktree for filesystem-level separation), **Remote** (internal-only, launching in a remote Claude Code Remote environment), and **In-process** (shares the filesystem but operates within an isolated conversation context). The subagents return only summary-only results to the parent agent, and the state is managed by appending new boundary and summary events, ensuring the transcript remains mostly append-only.

Anthropic's analysis confirms that the core design of Claude Code—the operational harness—is the primary architectural differentiator, establishing a model that prioritizes deterministic infrastructure over decision scaffolding. This design pattern, where the model performs reasoning and the surrounding harness handles enforcement, suggests that agentic coding tools may be converging toward operating-system-like abstractions, with the core agent loop functioning as the kernel and all other subsystems acting as the operating system.
