# claude-code-architecture-analysis

**Parent:** [[content/L1/claude-code-agent-architecture|claude-code-agent-architecture]] — Claude Code, an agentic coding tool, is designed around five core values (Authority, Safety, Reliability, Capability Amplification, Contextual Adaptability), which guide the implementation of architectural principles like deny-first evaluation and five-layer context compaction. It operates a core while-loop loop and relies on complex surrounding subsystems including a seven-mode permission system, four extensibility mechanisms, and an append-oriented session store.

Claude Code, an agentic coding tool, allows users to run shell commands, edit files, and call external services on behalf of the user. This study analyzes the comprehensive architecture of Claude Code by examining its publicly available TypeScript source code and by comparing it with OpenClaw, an independent open-source AI agent system. OpenClaw answers many of the same design questions but from a different deployment context. The analysis identifies five human values, philosophies, and needs that motivate the architecture: human decision authority, safety and security, reliable execution, capability amplification, and contextual adaptability. These values are then traced through thirteen design principles to specific implementation choices. Structurally, the core of Claude Code is a simple while-loop that calls the model, runs tools, and repeats the cycle. However, the majority of the code resides in surrounding subsystems, including a permission system with seven modes and an ML-based classifier, a five-layer compaction pipeline for context management, four extensibility mechanisms (Mechanical Composition Protocol/MCP, plugins, skills, and hooks), a subagent delegation and orchestration mechanism, and append-oriented session storage. A comparison of Claude Code with OpenClaw, a multi-channel personal assistant gateway, demonstrates that the same recurring design questions yield different architectural answers when the deployment context changes. Specifically, the shift is observed from per-action safety evaluation to perimeter-level access control, from a single Command Line Interface (CLI) loop to an embedded runtime within a gateway control plane, and from context-window extensions to gateway-wide capability registration. Finally, the authors identify six open design directions for future agent systems, which are grounded in recent empirical, architectural, and policy literature. 

*Self-Contained Background:* Claude Code is an agentic coding tool developed by Anthropic. Anthropic’s framework for safe agents identifies a central tension: agents must operate autonomously for maximum value, but humans must retain control over the pursuit of goals. Claude’s Constitution resolves this tension not through rigid decision procedures, but by cultivating “good judgment and sound values that can be applied contextually.” Empirical findings, including Anthropic’s internal survey of 132 engineers and researchers (Huang et al., 2025), suggest that the architecture enables qualitatively new workflows, with about 27% of Claude Code-assisted tasks being work that would not have been attempted otherwise. The full study is structured as follows: Section 2 identifies the five human values and thirteen design principles. Section 3 analyzes the high-level architecture and the design questions it answers. Sections 4 through 9 each analyze a major subsystem's specific design choices. Section 10 contrasts the findings with OpenClaw. Section 11 provides discussion, and Section 12 surveys six open questions for future agent systems. The core agent loop is a while-true cycle with state management, but subsystems for safety, extensibility, context management, delegation, and persistence constitute the bulk of the implementation. A running example tracks the task “Fix the failing test in auth.test.ts” through multiple architectural layers, including tool invocation, permission checks, context selection, iterative repair, delegation, and session persistence.

*Technical Detail Preservation:* The analysis utilizes source-level code analysis (supplemented by official Anthropic documentation) to identify design choices and trade-offs, rather than relying solely on product descriptions. This methodology allows the authors to build a deep understanding of the system mechanisms, aiming to inform the design of better and more powerful agent systems. The core systems identified for analysis are: a permission system with seven modes and an ML-based classifier; a five-layer compaction pipeline for context management; four extensibility mechanisms (MCP, plugins, skills, and hooks); a subagent delegation and orchestration mechanism; and append-oriented session storage. The open design directions identified are focused on the observability-evaluation gap, cross-session persistence, harness boundary evolution, horizon scaling, governance, and the evaluative lens.
