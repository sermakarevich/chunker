# agentic-rag-architectures-taxonomy

**Parent:** [[index]]

The integration of autonomous agents into Retrieval-Augmented Generation (RAG) systems creates Agentic RAG, which represents a significant advancement over traditional static RAG pipelines. Agentic RAG enables dynamic retrieval, iterative context refinement, and adaptive workflow orchestration, allowing the system to address complex, multi-domain tasks requiring both precision and adaptability. To illustrate this, the field proposes a structured view of Agentic RAG by introducing a principled taxonomy based on four design dimensions: agent cardinality, control structure, autonomy, and knowledge representation.

**Specialized Architectures:**

1. **Adaptive RAG:** This architecture enhances Large Language Models (LLMs) by dynamically adjusting the query handling strategy based on the complexity of the incoming query. The system uses a classifier to assess query complexity, determining the most appropriate process: **Straightforward Queries** (e.g., asking, "What is the boiling point of water?" which requires no additional retrieval, using pre-existing knowledge); **Simple Queries** (e.g., determining the status of a latest electricity bill, requiring a single-step retrieval); or **Complex Queries** (e.g., "How has the population of City X changed over the past decade, and what are the contributing factors?", requiring multi-step retrieval and progressive refinement). Key advantages of Adaptive RAG include Dynamic Adaptability, Resource Efficiency, Enhanced Accuracy, and Flexibility.

2. **Graph-Based Agentic RAG (Agent-G):** Agent-G enhances RAG by integrating graph knowledge bases with unstructured document retrieval. The core principle of Agent-G is the ability to dynamically assign retrieval tasks to specialized agents, utilizing both structured and unstructured data sources. The Agent-G system adjusts its retrieval strategy via four steps: (1) **Graph Knowledge Bases:** Extracting relationships and hierarchies from structured data (e.g., disease-to-symptom mappings in healthcare); (2) **Unstructured Documents:** Employing traditional text retrieval for contextual information; (3) **Critic Module:** Evaluating the relevance and quality of retrieved information to ensure alignment with the original query; and (4) **Feedback Loops:** Refining retrieval and synthesis through iterative validation and re-querying. The Agent-G system utilizes a Retriever Bank, a Critic Module (which flags low-confidence results), Dynamic Agent Interaction, and LLM Integration.

3. **Graph-Enhanced Agent for RAG (GeAR):** GeAR improves traditional RAG by incorporating graph-based retrieval mechanisms. GeAR advances RAG performance using two primary innovations: (1) **Graph Expansion:** This technique enhances conventional base retrievers (such as BM25) by expanding the retrieval process to include graph-structured data, enabling the system to capture complex relationships and dependencies between entities; and (2) **Agent Framework:** This framework uses the graph expansion to manage retrieval tasks, allowing for dynamic and autonomous decision-making. The GeAR workflow integrates a Graph Expansion Module, an Agent-Based Retrieval component, and LLM Integration.

**Other Specialized Architectures:**

*   **Multi-Agent Agentic RAG:** This architecture involves the parallel execution of multiple agents, enabling efficient processing of diverse query types. The workflow involves passing data from every agent to a Large Language Model (LLM) for synthesis, generating a comprehensive response. Advantages include Modularity, Scalability, Task Specialization, and Versatility.
*   **Hierarchical Agentic RAG Systems:** These systems use a structured, multi-tiered approach where a top-tier agent receives and assesses the query, then delegates specific tasks to lower-level agents specialized in methods (e.g., SQL databases, web search, proprietary systems). The higher-level agent finally aggregates and synthesizes the information for the user.
*   **Corrective RAG:** This method enables self-correction of retrieval results by embedding intelligent agents into the workflow. Corrective RAG ensures iterative refinement through a **Relevance Evaluation Agent** (assessing documents); a **Query Refinement Agent** (rewriting queries); an **External Knowledge Retrieval Agent** (performing web searches or accessing alternative data sources if context is insufficient); and a **Response Synthesis Agent** (generating the final, accurate response). This process improves factuality and accuracy by dynamically identifying and correcting irrelevant or ambiguous retrieval results.

**Practical Applications and Tools:**

Agentic RAG demonstrates transformative potential across sectors: 
*   **Customer Support:** Twitch used an agentic workflow on Amazon Bedrock to generate detailed ad proposals by integrating advertiser data, historical campaign performance, and audience demographics.
*   **Healthcare:** Systems integrate patient-specific data like electronic health records (EHR) and up-to-date medical literature to generate comprehensive patient case summaries and personalized care recommendations.
*   **Legal and Contract Analysis:** Systems use semantic search and legal knowledge graphs to analyze contracts, extract critical clauses, and identify potential risks.
*   **Finance and Risk Analysis:** Systems use real-time data streams and predictive modeling, such as in auto insurance claims processing, where the system combines policy details with accident data to generate recommendations while ensuring regulatory compliance.
*   **Education:** Agentic RAG supports adaptive learning.

For implementation, various tools support Agentic RAG: LangChain provides modular components, while LangGraph builds upon LangChain by introducing graph-based workflows supporting loops and self-correction. LlamaIndex's Agentic Document Workflows (ADW) enable end-to-end automation of document processing and structured reasoning. Other frameworks include CrewAI (supporting hierarchical and sequential processes), AutoGen (excelling in multi-agent collaboration), and OpenAI Swarm (for lightweight multi-agent orchestration). Furthermore, Microsoft developed Semantic Kernel, an open-source SDK that integrates Large Language Models (LLMs) into applications, and Google provided Vertex AI, which integrates with Agentic RAG for building scalable AI capabilities. Neo4j, an open-source graph database, can be paired with vector databases (like Weaviate, Pinecone, Milvus, and Qdrant) to form the backbone of high-performance Agentic RAG workflows.
