# orthogonal-systems-benefits

**Parent:** [[content/L1/orthogonality-design-principles-and-anti-patterns|orthogonality-design-principles-and-anti-patterns]] — The synthesis details the principles of orthogonality, outlining how component separation improves productivity and reduces risk, and discusses technical strategies for maintaining this separation in development and data handling.

Adopting orthogonal systems provides two major benefits: increased productivity and reduced risk. **Increased Productivity:** When using orthogonal systems, changes remain localized, which reduces both development time and testing time. Furthermore, it is easier to write small, self-contained components than to write a single, large block of code. Simple components can be designed, coded, tested, and then forgotten, eliminating the need to continuously change existing code when new code is added. An orthogonal approach also promotes reuse: if components have specific, well-defined responsibilities, developers can combine them with new components in ways that the original implementors of the components may not have envisioned. The looser the coupling of the system, the easier it becomes to reconfigure and reengineer the system. Specifically, combining orthogonal components yields more functionality per unit effort. For instance, if one component does distinct things and another component does distinct things, and the two components are combined while remaining orthogonal, the resulting system does distinct things. However, if the two components are not orthogonal, the combined system will suffer from overlap and will do less.
