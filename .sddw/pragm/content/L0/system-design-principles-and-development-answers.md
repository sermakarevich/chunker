# system-design-principles-and-development-answers

**Parent:** [[index]]

When analyzing technical problems, developers should consider the difference between tools with a graphical user interface (GUI) and small, combinable command-line utilities used at shell prompts. When deciding which set of tools is superior, developers must evaluate criteria such as orthogonality, ease of use for the intended purpose, combination ability with other tools to meet new challenges, and learning curve. Regarding system design, developers must adhere to the principle of decoupling, which requires keeping separate concepts separate through good design principles, as coupling hinders change. The following exercises provide conceptual answers to development problems:

*   **Answer 25 (from exercise 25):** When working with a list of key-value pairs, the key is assumed to be unique; hash libraries typically enforce this through the hash's behavior or explicit error messages. However, an array does not guarantee uniqueness and will happily store duplicate keys unless the developer codes it specifically not to. If duplicates exist in the array case, the first key found that matches the `DepositAccount` class wins, and any remaining matching entries are ignored. Furthermore, the order of entries is not guaranteed, which introduces variability in results.
*   **Answer 26 (from exercise 26):** A purely numeric field being functional in the US, Canada, and the Caribbean is a coincidence. According to the ITU specification, international call format must begin with a literal '+' sign. The '*' character is also used in some locales, and leading zeros can be part of the number. Therefore, a developer must never store a phone number in a numeric field.
