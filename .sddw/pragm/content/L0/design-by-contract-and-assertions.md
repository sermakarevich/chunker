# design-by-contract-and-assertions

**Parent:** [[content/L1/advanced-software-development-methodologies|advanced-software-development-methodologies]] — Professional development necessitates adopting principles like DRY, Design by Contract (DBC) which formalizes function expectations into Preconditions, Postconditions, and Class Invariants, and continuous process refinement using methods like PERT and the Tracer Bullet approach.

Design by Contract (DBC) is a technique that applies formal contracts to software modules to ensure program correctness. DBC mandates that developers document and verify the expected behavior of every function or method, ensuring that the code performs no more and no less than what it claims to do. DBC offers a significant advantage over defensive programming, which requires manual validation of data everywhere.
