# design-by-contract-advanced-topics

**Parent:** [[content/L1/software-design-principles-and-process|software-design-principles-and-process]] — Software development requires adherence to principles like DRY, which mandates single, authoritative knowledge representations, and employing techniques such as PERT (optimistic, most likely, pessimistic times) and 'Eating the Elephant' (iterative thin slices) to manage uncertainty.

Regarding Design by Contract (DBC), the core questions to ponder are: If DBC is so powerful, why is it not used more widely? Is it difficult to formulate the contract? Does adopting DBC force a developer to think about issues that the developer would rather ignore in the short term? If DBC forces the developer to think, it is considered a powerful, and perhaps dangerous, tool. For example, when designing an interface for a kitchen blender—which will eventually be web-based and IoT-enabled—but currently only requires the interface to control it, developers must determine appropriate pre- and postconditions and invariants for the following methods: `int getSpeed()`, `void setSpeed(int x)`, `boolean isFull()`, `void fill()`, and `void empty()`. Furthermore, developers should be able to solve problems like counting the number of integers in the series 0, 5, 10, 15, $\ldots$, 100. When program failures occur, recognizing that software defects (or bugs) are inherent because no one writes perfectly functioning software is critical. Sometimes, other people (or systems) can detect issues with a program before the developer becomes aware of the problem. Examples include receiving a `nil` value, finding an empty list, encountering a missing key in a hash, or determining that a value expected to be a hash is actually a list. Other detectable failures include network or filesystem errors resulting in empty or corrupted data, or a logic error that, occurring millions of instructions after the root cause, changes the expected value of a case statement selector, causing the program to unexpectedly hit a default case.
