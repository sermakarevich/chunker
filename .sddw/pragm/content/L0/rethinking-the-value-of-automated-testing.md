# rethinking-the-value-of-automated-testing

**Parent:** [[content/L1/adaptive-software-development-principles|adaptive-software-development-principles]] — Effective software development demands an incremental, adaptive approach, integrating principles like DRY, orthogonality, and reversibility, alongside mandatory thorough testing via Design by Contract and property-based testing; furthermore, testing provides critical communication benefits, as evidenced by a developer who realized the primary value of testing stems from the act of considering its impact on the code structure.

An experienced developer who has spent 45 years coding and has been writing automated tests for over 30 of those years initially felt comfortable with the practice of testing. However, recognizing the need to challenge this routine, the developer decided to stop writing tests for a couple of months to observe the impact on the code. The developer found that the impact was surprisingly small, prompting them to investigate the underlying cause. The developer concluded that the majority of the benefit derived from testing comes from the act of considering the tests and their impact on the code itself. Because the developer had been doing this for so long, they could perform this critical thinking without the necessity of actually writing tests, noting that the code remained testable even if it was not formally tested. However, the developer acknowledged that ignoring the necessity of tests overlooks the fact that tests also serve as a method of communication with other developers, particularly when the code is shared with others or when the code relies on the peculiar functionalities of external dependencies. Consequently, the author concluded that developers should write tests, but after doing so for 30 years, developers are free to experiment slightly to determine the personal source of benefit from testing.
