# orthogonality-design-principles-and-anti-patterns

**Parent:** [[content/L2/software-architecture-methodologies|software-architecture-methodologies]] — Software development requires adhering to principles like DRY (Don't Repeat Yourself) and orthogonality, ensuring knowledge is singularly represented and modules function independently; complex projects must use range-based scheduling techniques (PERT) and iterative 'Eating the Elephant' approaches, while development should utilize continuous 'Tracer Code' for visible, measurable progress rather than disposable prototypes.

The foundational principles of good software design mandate adherence to DRY (Don't Repeat Yourself) and promoting highly orthogonal system structures. Orthogonality, in this context, signifies that a system is composed of cooperating modules, where each module functions independently of the others. Adopting this approach yields two major benefits: increased productivity and significantly reduced risk. From a productivity standpoint, when components are orthogonal, changes remain localized, which directly reduces both the development time and the time required for thorough testing.
