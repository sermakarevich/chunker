# iterative-feedback-loop-in-software-design

**Parent:** [[index]]

To ensure successful software development, teams must continuously manage uncertainty. Instead of relying on a fixed plan, the team should actively gather and act on feedback, following a three-step iterative loop: 1) identify the current status, 2) make the smallest meaningful step toward the desired goal, and 3) evaluate the result to fix any broken components. Teams should repeat these steps recursively at every level of work. For instance, improving a minor detail, such as renaming a variable from `user` to `owner` and optimizing it to `email` in the code example `let email = emailOfAccountOwner(accountID);`, can significantly improve the overall system design by reducing the coupling between the code and account management functionality. Furthermore, this feedback loop must apply even at the highest level of a project; successful work occurs when a team starts with a client’s requirements, takes a single step, and realizes that the best solution may require no software at all. This iterative feedback loop must also apply to reviewing the team's process. Furthermore, because fixing broken components (Step 3) requires good design, the team should focus on making things easier to change, a necessity that defines agility. Beyond the software process, the simple feedback loop can improve any decision; the goal is to consider ways to undo recent decisions if the desired direction changes.
