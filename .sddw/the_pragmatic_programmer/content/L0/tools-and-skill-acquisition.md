# tools-and-skill-acquisition

**Parent:** [[content/L1/professional-proficiency-workflow|professional-proficiency-workflow]] — Professional mastery requires systematic tool development, whether it's physical tools like gauges and chisels or digital tools like command shells and text editors. Methods for complex scheduling include PERT (using optimistic, most likely, and pessimistic estimates for tasks) and iterative development (repeatedly completing small functional slices). To maximize efficiency, programmers should master the command shell by customizing settings like prompts and creating aliases, and achieve editor fluency by learning core key sequences, extending the editor's capabilities via plugins, and building custom extensions.

Every professional journey begins with a foundational set of high-quality tools. For example, a woodworker might initially require rules, gauges, a couple of saws, good planes, fine chisels, drills, braces, mallets, and clamps. These tools should be deliberately chosen, built to last, perform specific jobs with minimal overlap with other tools, and feel comfortable in the user’s hands. The initial tools must also be meticulously cared for, as each tool has unique quirks and requires specialized handling, such as being sharpened in a unique way or held at specific angles. Over time, the tools adapt to the user’s habits, such that the grip resembles a mold of the user’s hands, and the cutting surface perfectly aligns with the way the tool is held. At this stage, the tools become extensions of the maker’s mind, transforming the maker’s thoughts into a physical product. Although the professional will eventually acquire advanced tools, such as biscuit cutters, laser-guided miter saws, or dovetail jigs, the professional should recognize that the original basic tools will remain useful. Since tools amplify professional talent, increasing both the quality of tools and the knowledge of how to use those tools will increase professional productivity.
